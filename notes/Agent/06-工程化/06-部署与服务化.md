# 部署与服务化

## 本笔记的学习目标
- 理解 Agent 从本地原型到线上服务需要解决的部署、运行和运维问题。

## 核心内容
- Agent 服务化不是把脚本包成 API，而是要处理鉴权、任务队列、状态持久化、异步执行、超时、观测、成本和安全。
- Microsoft 课程的 Production 主题强调可扩展、可信、可运维的 Agent；企业平台通常会提供状态管理、权限和观测能力。
- 长任务 Agent 往往需要异步任务模型，而不是同步 HTTP 请求一直等待。

## 部署形态
- 本地脚本：适合学习和原型。
- API 服务：适合短任务和产品集成。
- 后台任务：适合长时间研究、批处理和文件处理。
- 工作流服务：适合多步骤业务流程。
- 托管平台：适合需要权限、审计、扩展和企业集成的场景。

## 工程化要点
- 密钥使用环境变量或密钥管理服务，不写入代码和笔记。
- 长任务要支持任务 ID、进度查询、取消和恢复。
- 模型和工具调用要设置超时。
- 部署环境需要日志、指标、告警和成本监控。
- 对外服务要有鉴权、限流、输入校验和权限控制。

## 上线检查
- 是否有评估集和回归测试。
- 是否有 Trace 和错误日志。
- 是否有成本上限。
- 是否有安全策略和人工确认。
- 是否支持失败恢复。
- 是否明确用户数据保留策略。
