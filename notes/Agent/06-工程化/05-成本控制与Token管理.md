# 成本控制与Token管理

## 本笔记的学习目标
- 理解 Agent 系统中 Token、模型调用次数、工具调用和基础设施成本的控制方法。

## 核心内容
- Agent 的成本来自模型输入输出 Token、工具调用、检索、重排、存储、部署和人工审核。
- Agent 比普通 LLM 应用更容易成本失控，因为它可能多轮调用模型、多次检索、多次工具执行。
- 成本控制不是单纯换便宜模型，而是通过上下文管理、模型路由、缓存、停止条件和评估来优化。

## 成本来源
- 长上下文输入：历史消息、检索片段、工具结果过多。
- 多轮循环：Planning、Reflection、工具调用反复进行。
- 强模型滥用：简单任务也走高成本模型。
- 检索和重排：RAG 管线额外带来延迟和费用。
- 失败重试：错误处理不当会放大调用次数。

## 优化策略
- 上下文裁剪：只保留当前任务相关信息。
- 摘要压缩：长期历史用摘要和结构化状态代替原文。
- 模型路由：简单任务用小模型，复杂任务升级。
- 缓存：缓存稳定 Prompt、检索结果和常见答案。
- 限制循环：设置最大步数、最大工具调用次数和最大成本。
- 批处理：离线任务批量执行，降低峰值成本。

## 工程化理解
- 成本指标要按任务、用户、模型、工具和功能拆分。
- 每个 Agent 任务都应有预算上限。
- 质量和成本要一起评估，不能单看价格。
