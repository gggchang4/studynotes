# 错误处理、重试与降级

## 本笔记的学习目标
- 掌握 Agent 系统中常见错误的分类、处理、重试和降级策略。

## 核心内容
- Agent 的错误来源比普通应用更多：模型可能误解任务，工具可能失败，RAG 可能召回错误，结构化输出可能不合法，状态可能不一致。
- 错误处理不是简单 try-catch，而是设计系统在失败时如何恢复、降级、请求用户补充或停止。

## 错误分类
- 模型错误：幻觉、格式错误、工具选择错误、拒答不当。
- 工具错误：参数错误、超时、限流、权限不足、外部服务失败。
- RAG 错误：召回为空、召回无关、引用错误、权限过滤失败。
- 状态错误：步骤重复、状态丢失、并发冲突、恢复失败。
- 安全错误：Prompt Injection、越权工具调用、敏感数据泄露。
- 产品错误：用户目标不明确、需求冲突、结果不可用。

## 处理策略
- 重试：适合临时网络错误、限流和格式修复。
- 修复：适合结构化输出不合法、参数缺失。
- 降级：切换模型、减少工具、返回部分结果或转人工。
- 询问：用户输入不足或约束冲突时请求澄清。
- 停止：达到最大步数、风险过高或失败不可恢复时终止。

## 工程化理解
- 重试必须有上限和退避策略。
- 同类错误反复出现应进入评估集，而不是只在代码里打补丁。
- 降级结果要明确告诉上层系统，不要伪装成完整成功。
- 高风险失败要保留完整 Trace，便于审计。
