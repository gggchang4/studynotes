# ComplyX法律合规Agent

## 本笔记的学习目标
- 设计一个面向法律合规场景的 Agent 项目雏形，强调 RAG、风险识别、引用溯源和人工复核。

## 项目目标
- 构建一个能基于法规、政策、合同或内部制度进行合规问答、风险初筛和审查辅助的 Agent。
- 该 Agent 不能替代律师或合规负责人，只能作为辅助系统。

## 推荐功能范围
- 法规和制度文档知识库。
- 合规问答：基于引用回答问题。
- 风险识别：对输入文本标记潜在风险点。
- 条款匹配：找到相关法规或制度条款。
- 审查报告草稿：输出问题、依据、风险等级、建议和待人工确认事项。
- 人工复核：所有高风险结论必须进入人工确认。

## 架构设计
- RAG 层：法规文档解析、条款级 Chunk、元数据和引用。
- Agent 层：识别任务类型，选择问答、审查或报告流程。
- 评估层：事实一致性、引用准确性、风险漏报和误报。
- 安全层：敏感数据处理、权限过滤、审计日志和免责声明。

## 风险边界
- 不输出确定法律结论，只输出辅助判断和依据。
- 对缺少依据的结论必须明确说明不确定。
- 保留所有引用来源，便于人工复核。
- 用户上传材料要考虑隐私和数据保留策略。
