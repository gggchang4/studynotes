# RAG知识库问答Agent

## 本笔记的学习目标
- 设计一个基于 RAG 的知识库问答 Agent，支持文档检索、引用溯源和答案评估。

## 项目目标
- 构建一个能基于私有文档回答问题的 Agent。
- 答案必须尽量基于检索内容，不能凭空编造。
- 输出需要包含引用来源，便于复查。

## 推荐功能范围
- 支持 Markdown、PDF 或网页文档入库。
- 支持 Chunk 切分、Embedding、向量检索和 Rerank。
- 支持 Query Rewrite。
- 支持答案引用来源。
- 支持“找不到依据时拒答或说明不足”。
- 支持 RAG 评估集和基础指标。

## 实现步骤
- 选择文档集合，确定文档权限和元数据。
- 实现文档解析和 Chunk 切分。
- 生成 Embedding 并写入向量数据库。
- 实现 Retriever 和 Rerank。
- 设计 RAG Prompt：要求只基于上下文回答并输出引用。
- 构建评估集：事实问答、跨文档问答、无答案问题、权限问题。
- 记录检索和生成 Trace。

## 验收标准
- 能召回相关文档片段。
- 答案引用能定位到来源。
- 无依据问题不会强行回答。
- 检索和生成过程可复盘。
- 文档更新后可重新索引。

## 后续需要补充的知识点清单

- RAG Prompt 模板
- 引用格式
- RAG 评估集
