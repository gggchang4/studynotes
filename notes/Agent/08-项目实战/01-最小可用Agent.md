# 最小可用Agent

## 本笔记的学习目标
- 设计并实现一个最小可用 Agent，用来验证 Agent 核心循环、工具调用、状态记录和评估方式。

## 项目目标
- 构建一个能接收用户任务、判断是否需要工具、调用一个简单工具、整合结果并返回答案的 Agent。
- 目标不是功能复杂，而是把 Agent 的最小闭环跑通：输入、思考、行动、观察、输出、日志。

## 推荐功能范围
- 支持一个简单工具，例如计算器、搜索模拟器、文件读取模拟器。
- 支持结构化工具调用参数。
- 支持最大步数和停止条件。
- 支持基础 Trace：记录模型输入、工具调用、工具结果和最终输出。
- 支持 5 到 10 条最小评估用例。

## 实现步骤
- 定义任务边界：Agent 能做什么，不能做什么。
- 设计 System Prompt：说明角色、工具规则、输出格式和停止条件。
- 定义 Tool Schema：工具名、参数、返回结构和错误信息。
- 实现 Agent Loop：Think -> Act -> Observe -> Stop。
- 添加日志：记录每一步输入输出。
- 建立评估集：正常问题、工具参数错误、工具失败、无需工具问题。

## 验收标准
- Agent 能正确判断是否需要工具。
- 工具参数能通过校验。
- 工具失败时能给出可理解的处理。
- 不会无限循环。
- 每次运行能复盘完整执行过程。
