# 工具调用型Agent

## 本笔记的学习目标
- 设计一个能安全调用外部 API 或业务工具的 Agent 项目。

## 项目目标
- 构建一个面向真实外部系统的工具调用 Agent，例如天气查询、日程管理、工单查询、数据库查询或邮件草稿生成。
- 重点验证工具 Schema、参数校验、权限控制、错误处理和审计日志。

## 推荐功能范围
- 至少接入 2 到 3 个工具。
- 工具调用前做参数校验。
- 工具调用后返回结构化结果。
- 高风险工具只生成草稿或等待人工确认。
- 支持工具失败时重试、降级或询问用户。
- 记录工具调用审计日志。

## 实现步骤
- 列出可用工具和禁止操作。
- 为每个工具设计 Schema、描述、参数约束和返回格式。
- 设计工具选择 Prompt。
- 实现工具执行器和权限检查。
- 实现错误处理和重试。
- 建立 BFCL 风格的工具调用评估集。

## 验收标准
- Agent 能选对工具。
- 工具参数正确率可评估。
- 越权或危险请求被拒绝。
- 工具失败时不会编造成功结果。
- 所有工具调用可审计。
