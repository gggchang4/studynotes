# 多步骤任务Agent

## 本笔记的学习目标
- 设计一个能够规划、执行、检查并完成多步骤任务的 Agent。

## 项目目标
- 构建一个能处理复杂目标的 Agent，例如“调研一个主题并生成摘要”“分析一个代码库并给出修复建议”“规划一趟旅行”。
- 重点验证 Planning、State、Tool Calling、Reflection 和 Workflow 编排。

## 推荐功能范围
- 支持生成结构化计划。
- 支持逐步执行计划。
- 支持中间结果写入状态。
- 支持失败后重规划。
- 支持最终结果检查。
- 支持人工确认关键步骤。

## 实现步骤
- 定义任务类型和完成标准。
- 设计计划 Schema：步骤、工具、输入、输出、验收条件。
- 实现 Planner 和 Executor。
- 实现状态管理和步骤记录。
- 接入工具，例如搜索、文件读取、代码分析或数据库查询。
- 添加 Reflection 节点检查结果。
- 建立多步骤任务评估集。

## 验收标准
- 计划步骤完整且可执行。
- 每一步状态可追踪。
- 工具失败后能恢复或停止。
- 最终输出能对应原始目标。
- 复杂任务不会无限循环。
