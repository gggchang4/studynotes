# Memory记忆机制

## 本笔记的学习目标
- 理解 Agent 记忆的类型、作用、存储方式和安全边界。

## 核心内容
- Memory 是 Agent 跨步骤、跨会话或跨任务保留信息的机制，不能简单等同于聊天历史。
- Microsoft 课程把 Agentic Memory 作为独立主题，Datawhale 课程把记忆与检索、上下文工程放在一起讲，说明 Memory 与 RAG、Context 有强关联。
- 短期记忆保存当前任务上下文，例如当前计划、已完成步骤、工具结果。
- 长期记忆保存跨会话信息，例如用户偏好、项目知识、稳定事实、历史决策。
- 情景记忆保存过去执行轨迹，语义记忆保存抽象知识，程序性记忆保存可复用技能或操作步骤。

## Memory 与 RAG 的区别
- RAG 通常面向外部知识库，目标是按问题检索相关资料。
- Memory 通常面向 Agent 自身和用户历史，目标是让系统连续、个性化、可积累。
- 二者可以共用向量数据库，但数据来源、更新策略、权限和生命周期不同。

## 工程化要点
- 记忆写入必须有策略，不能把所有对话都永久保存。
- 长期记忆需要可查看、可删除、可纠错。
- 敏感信息不应默认进入长期记忆。
- 记忆检索结果应该带来源、时间和置信度。
- Memory 写入和读取都应该进入 Trace，便于排查错误个性化。
