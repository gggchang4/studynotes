# Reasoning与Reflection

## 本笔记的学习目标
- 理解推理、反思和元认知在 Agent 决策质量中的作用，以及如何工程化地使用它们。

## 核心内容
- Reasoning 是模型围绕任务目标进行判断、比较、分解和选择的过程。
- Reflection 是 Agent 在输出或行动后检查结果、发现问题、修正计划或重试的机制。
- Microsoft 的 Metacognition 设计模式强调让 Agent 评估自己的理解、置信度和下一步策略。
- Reflection 不等于让模型无限自我批评，而是围绕明确标准进行有限次检查。

## 常见用法
- 输出前检查：检查是否满足用户目标、格式、事实依据和约束。
- 工具结果检查：判断工具结果是否成功、是否需要重试或换工具。
- 计划复盘：执行失败后分析原因并重新规划。
- 多候选比较：生成多个方案后用评估器或 Judge 选择。
- 低置信度升级：当模型不确定时请求更多信息或人工确认。

## 工程化要点
- Reflection 必须有次数上限，避免成本失控和循环。
- 反思标准要具体，例如“是否有引用”“金额是否一致”“工具参数是否完整”。
- 不要把隐藏推理过程作为业务依据，应该记录可审计的决策摘要和证据。
- 关键任务需要外部评估器、规则校验或人工复核，而不是只靠模型自评。
