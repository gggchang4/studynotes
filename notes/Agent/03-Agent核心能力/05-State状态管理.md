# State状态管理

## 本笔记的学习目标
- 理解 Agent 执行过程中的状态建模、状态传递、状态恢复和持久化。

## 核心内容
- State 是当前任务执行过程中需要持续维护的结构化信息，例如目标、计划、当前步骤、工具结果、错误、用户确认、输出草稿。
- Memory 关注跨任务长期信息，State 关注当前任务运行时信息。
- 在 LangGraph 等图编排框架中，State 是节点之间传递信息的核心对象。
- 对生产 Agent 来说，状态管理决定了能否暂停、恢复、重试、审计和人工接管。

## 状态类型
- 会话状态：用户、会话 ID、当前上下文、权限范围。
- 任务状态：任务目标、计划、步骤进度、完成条件。
- 工具状态：调用记录、结果、错误、重试次数。
- 业务状态：订单、文档、审批、合规风险等业务实体。
- 控制状态：是否终止、是否等待用户、是否升级人工、是否降级模型。

## 工程化要点
- 状态应该结构化，不要只存在聊天历史里。
- 每次工具调用后要更新状态，而不是只让模型“记住”。
- 长任务需要 checkpoint，支持失败后恢复。
- 状态变更要能追踪，关键字段要有来源。
- 状态中不能混入未校验的不可信指令。
