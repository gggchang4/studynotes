# Tool Calling与外部系统

## 本笔记的学习目标
- 理解 Agent 如何通过 Tool Calling / Function Calling 调用外部系统，并掌握工具设计、安全和错误处理要点。

## 核心内容
- Tool Calling 是让模型选择并生成工具调用参数，由程序执行真实操作，再把结果返回给模型的机制。
- Function Calling 更强调模型按函数 Schema 输出参数；Tool Calling 可以包括 API、数据库、搜索、文件、浏览器、代码执行和业务动作。
- Hugging Face 课程在基础单元和 Bonus Unit 中讲到 Tools 与 Function Calling；Microsoft 课程把 Tool Use 作为核心设计模式。
- 工具是 Agent 的“手”，也是风险最大的部分，因为它可能改变外部世界。

## 工具设计原则
- 工具名清晰：模型能从名称理解用途。
- 描述具体：说明什么时候用、什么时候不用。
- 参数可校验：使用类型、枚举、范围、必填字段。
- 结果结构化：返回状态码、关键字段、错误信息和可读摘要。
- 单一职责：一个工具做一类事，不要混合太多隐式行为。
- 最小权限：工具只能访问完成任务所需的资源。

## 工程化要点
- 工具调用前校验参数，调用后校验结果。
- 高风险工具需要人工确认，例如删除、支付、发邮件、改权限。
- 工具错误要可恢复，例如重试、换工具、请求用户补充信息。
- 工具调用必须被记录，包括输入参数、输出结果、耗时、错误和调用者。
- 不可信工具结果不能当作系统指令，应作为数据进入上下文。
