# Agent知识库总览

## 本笔记的学习目标
- 建立「AI Agent 搭建与工程化」知识库的全局视角，知道各目录负责什么、学习时从哪里进入、项目实践时如何查找资料。

## 核心内容
- 本知识库参考三类课程材料：Hugging Face 偏动手实践与框架体验，Datawhale 偏中文系统教材与项目案例，Microsoft 偏设计模式、协议、安全和生产化。
- 知识库的主线不是按课程章节照搬，而是按真实工程开发顺序组织：先理解 Agent，再掌握 LLM 应用基础，然后学习工具、记忆、规划、RAG、编排，最后进入评估、安全、部署和项目实战。
- `01-基础概念` 负责回答「什么是 Agent、它和 Chatbot/Workflow 有什么不同、什么时候该用」。
- `02-LLM应用基础` 负责 Prompt、上下文、结构化输出、模型选择等通用 LLM 应用能力。
- `03-Agent核心能力` 负责 Planning、Reasoning、Memory、Tool Calling、State、Multi-Agent。
- `04-RAG与知识库` 负责文档入库、检索、重排、Query Rewrite、评估和工程化。
- `05-Agent框架与编排` 负责 LangGraph、Workflow、MCP 等框架和协议层能力。
- `06-工程化` 负责项目架构、评估、可观测性、错误处理、成本、安全和部署。
- `08-项目实战` 负责把知识压到真实项目里，包括最小 Agent、RAG Agent、工具调用 Agent、多步骤 Agent、合规 Agent 和 Web Agent。

## 工程化理解
- Agent 不是一个单独算法，而是一个由模型、工具、状态、记忆、权限、评估和运行环境共同组成的系统。
- 搭建 Agent 时优先明确任务边界，再决定是否需要工具、RAG、记忆、多智能体或图编排。
- 工程落地的关键不是让 Agent「看起来聪明」，而是让它可控、可测、可观测、可恢复、可审计。
