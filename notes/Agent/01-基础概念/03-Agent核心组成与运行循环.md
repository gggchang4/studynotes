# Agent核心组成与运行循环

## 本笔记的学习目标
- 理解 Agent 的核心模块，以及 Thought-Action-Observation / ReAct 循环如何驱动 Agent 完成任务。

## 核心组成
- 模型：负责理解任务、推理、生成计划和选择工具。
- Prompt 与上下文：定义角色、目标、约束、可用工具、历史信息和输出格式。
- 工具：让 Agent 访问外部世界，例如搜索、数据库、文件系统、浏览器、业务 API。
- 状态：保存当前任务进度、中间结果、已调用工具、错误和下一步动作。
- 记忆：保存跨会话或长期可复用的信息，例如用户偏好、项目背景、历史决策。
- 评估与可观测性：记录 Agent 是否做对、为什么错、哪里耗时、哪里失败。
- 权限与治理：限制 Agent 能做什么、什么时候需要人工确认、哪些数据不能暴露。

## 运行循环
- Observe：接收用户目标、环境反馈、工具结果或外部事件。
- Think：基于当前上下文判断下一步，需要时进行计划、推理或反思。
- Act：调用工具、生成结构化输出、发起子任务或返回结果。
- Observe Again：把行动结果重新纳入上下文，继续判断是否完成。
- Stop：满足目标、达到最大步数、遇到不可恢复错误或触发人工接管。

## 工程化理解
- 运行循环必须有停止条件，否则容易出现无限循环、重复调用和成本失控。
- 工具结果应该进入结构化状态，而不是只拼回自然语言历史。
- 每一次 Think 和 Act 都应该能被 Trace 复盘，否则后续很难评估和优化。
