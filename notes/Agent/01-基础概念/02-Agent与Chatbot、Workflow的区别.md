# Agent与Chatbot、Workflow的区别

## 本笔记的学习目标
- 区分 Chatbot、Workflow 和 Agent，避免在不需要 Agent 的地方过度设计。

## 核心内容
- Chatbot 的核心是对话交互，通常围绕用户输入生成回答，控制流比较简单。
- Workflow 的核心是确定性流程，步骤、分支和依赖关系通常由开发者提前定义。
- Agent 的核心是动态决策，模型会根据目标、上下文和工具结果决定下一步做什么。
- 三者不是互斥关系：一个产品可以用 Chatbot 作为交互界面，用 Workflow 管理主流程，在某些节点调用 Agent 处理开放性子任务。

## 对比表
| 维度 | Chatbot | Workflow | Agent |
|---|---|---|---|
| 控制流 | 对话驱动 | 开发者定义 | 模型参与决策 |
| 工具调用 | 可选 | 明确节点 | 动态选择 |
| 可预测性 | 中等 | 高 | 中低 |
| 适用任务 | 问答、客服、助手 | 审批、固定流程、批处理 | 研究、分析、排障、多步骤操作 |
| 工程风险 | 幻觉与误答 | 流程遗漏 | 越权、循环、成本、不可控行动 |

## 工程化理解
- 能用 Workflow 解决的任务，优先用 Workflow；只有当流程无法提前穷举时，再引入 Agent。
- Agent 最适合放在 Workflow 的某个节点中，作为“智能决策模块”，而不是替代所有流程控制。
- 在生产系统里，Agent 的自由度越高，越需要评估、日志、权限和人工确认。
