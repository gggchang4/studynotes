# Agent适合与不适合的场景

## 本笔记的学习目标
- 判断某个任务是否值得使用 Agent，并识别高风险或低收益场景。

## 适合的场景
- 多步骤任务：任务需要拆解、执行、检查和调整，例如调研报告、代码排障、旅行规划。
- 信息不完整：需要边查边做，根据新信息更新计划。
- 工具组合：需要调用搜索、数据库、文件、浏览器、业务 API 等多个工具。
- 个性化持续任务：需要记住用户偏好、项目背景或长期目标。
- 探索型任务：没有固定答案，需要比较、归纳、生成方案。

## 不适合的场景
- 流程高度确定：固定审批、固定表单处理、简单 ETL 更适合 Workflow。
- 高风险自动执行：转账、删除数据、发合同、改权限等必须有人审或强约束。
- 低延迟强一致任务：实时交易、严格确定性计算不适合交给 Agent 自由决策。
- 信息源不可信：如果输入容易被 Prompt Injection 污染，需要先做安全设计。
- 成本收益不匹配：单轮分类、摘要、抽取任务通常不需要完整 Agent。

## 决策清单
- 任务是否必须多步执行？
- 下一步是否依赖上一步结果？
- 是否需要调用外部工具？
- 是否需要记住长期背景？
- 是否允许一定不确定性？
- 是否有评估和人工复核机制？
