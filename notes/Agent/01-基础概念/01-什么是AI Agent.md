# 什么是AI Agent

## 本笔记的学习目标
- 理解 AI Agent 的定义、能力来源、边界和常见误区。

## 核心内容
- AI Agent 是一个能围绕目标进行感知、推理、行动和反馈调整的系统。它通常由 LLM、工具、上下文、状态、记忆和执行环境共同组成。
- 普通 LLM 主要完成文本生成，而 Agent 会把模型输出接入外部世界，例如调用 API、检索知识库、操作文件、浏览网页或协调多个子任务。
- Agent 的关键不是“完全自主”，而是“在受控边界内动态决策”。越接近真实生产环境，越需要限制权限、记录执行轨迹、设置人工确认和失败恢复机制。
- Hugging Face 课程强调 Thought-Action-Observation 循环，Microsoft 课程强调 Agent 是连接工具、知识和记忆的系统，Datawhale 课程强调从范式、框架到项目的逐步构建。

## 判断标准
- 是否需要多步骤决策：如果任务只是单轮回答，通常不需要 Agent。
- 是否需要外部行动：如果要调用工具、读写数据、执行操作，Agent 才有明显价值。
- 是否需要动态调整：如果流程固定，用 Workflow 更稳定；如果步骤依赖中间结果，Agent 更合适。
- 是否能承受不确定性：Agent 适合探索性任务，不适合无监督执行高风险动作。

## 工程化理解
- Agent 是系统，不是单个 Prompt。
- Agent 的能力上限由模型决定，可靠性由工程约束决定。
- 一个可上线 Agent 至少需要输入约束、工具权限、状态记录、错误处理、评估集和可观测性。
