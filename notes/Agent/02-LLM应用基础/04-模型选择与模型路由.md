# 模型选择与模型路由

## 本笔记的学习目标
- 理解不同模型在能力、成本、延迟、上下文长度和稳定性上的取舍，并设计可落地的模型路由策略。

## 核心内容
- 模型选择不是只看榜单分数，还要看任务类型、输出稳定性、工具调用能力、结构化输出能力、长上下文质量、价格、延迟和供应商稳定性。
- Agent 系统通常需要多模型协作：强模型负责复杂规划和高风险判断，小模型负责分类、摘要、改写、路由和低风险任务。
- 模型路由是根据任务复杂度、用户等级、成本预算、实时性和失败情况选择合适模型。
- 路由策略需要被评估集验证，否则容易出现“便宜模型省钱但质量不可控”或“强模型全包导致成本失控”。

## 路由策略
- 按任务路由：分类、摘要、抽取用小模型，规划、代码、复杂推理用强模型。
- 按风险路由：涉及权限、法律、财务、用户隐私的任务使用强模型并要求人工确认。
- 按上下文路由：长文档任务选择长上下文模型或先 RAG 压缩。
- 按失败路由：低成本模型失败后升级到强模型。
- 按成本预算路由：设置单任务最大 Token 和最大调用次数。

## 工程化理解
- 模型路由需要记录选择原因、模型版本、输入输出和结果质量。
- 同一 Prompt 换模型可能表现不同，需要独立评估。
- 不要把模型能力当成安全机制，权限控制和业务校验必须在模型外部完成。

- 多模型路由规则
- 降级策略和缓存策略
