# 第03章 数据库语言：开卷考试解题思路与模板

> 适用范围：SQL 语言简介、DDL 与基本数据库对象、DML 查询与更新、视图/索引、DCL 权限控制。  
> 课件常用关系模式：`Student(sno, sname, sex, age, dept)`，`Course(cno, cname, credit, pcno)`，`SC(sno, cno, grade)`。

## 0. 考场快速判题地图

| 题目关键词 | SQL 类型 | 先写什么 |
| --- | --- | --- |
| 创建、删除、修改表/模式/索引/视图/域 | DDL | `CREATE` / `DROP` / `ALTER` |
| 查询、求平均、统计、分组、排序、嵌套 | DML 查询 | `SELECT ... FROM ... WHERE ...` |
| 插入、修改、删除记录 | DML 更新 | `INSERT` / `UPDATE` / `DELETE` |
| 视图、虚表、只给部分用户看 | 视图 | `CREATE VIEW ... AS SELECT ...` |
| 授权、回收权限、角色 | DCL | `GRANT` / `REVOKE` / `CREATE ROLE` |

**通用解题顺序：**

1. 先圈出“目标列”：题目问输出什么，就放进 `SELECT`。
2. 再圈出“数据来源”：涉及哪些表，就放进 `FROM`。
3. 再圈出“普通条件”：行级筛选写进 `WHERE`。
4. 出现“每个/按/分组/分别统计”：加 `GROUP BY`。
5. 条件里出现聚合结果，如平均分大于 80、人数超过 30：写进 `HAVING`。
6. 出现“按...升序/降序”：最后写 `ORDER BY`。

```sql
SELECT [DISTINCT] 目标列或表达式 [AS 别名], ...
FROM 表名 [AS 别名], ...
WHERE 行条件
GROUP BY 分组列
HAVING 组条件
ORDER BY 排序列 [ASC | DESC];
```

## 1. SQL 语言简介

### 1.1 SQL 功能分类

| 功能 | 典型语句 | 考点 |
| --- | --- | --- |
| 数据定义 DDL | `CREATE`，`DROP`，`ALTER` | 模式、表、视图、索引、域 |
| 数据查询 | `SELECT` | 本章最核心 |
| 数据操纵 DML | `INSERT`，`UPDATE`，`DELETE` | 插入、修改、删除元组 |
| 数据控制 DCL | `GRANT`，`REVOKE` | 权限授予与回收 |

### 1.2 SQL 基本特点

- 综合统一：DDL、DML、DCL 集成在一套语言里。
- 高度非过程化：只说明“做什么”，不写“怎么做”。
- 面向集合：输入输出通常都是关系或元组集合。
- 使用方式灵活：可交互使用，也可嵌入高级语言。
- 核心动词少：`CREATE`、`DROP`、`ALTER`、`SELECT`、`INSERT`、`UPDATE`、`DELETE`、`GRANT`、`REVOKE`。

## 2. DDL：模式、表、完整性、域、索引

### 2.1 模式 Schema

**题型口令：**“创建一个数据库模式/命名空间，并指定拥有者”。

```sql
CREATE SCHEMA 模式名 AUTHORIZATION 用户名;
```

可在创建模式时同时创建表、视图、域：

```sql
CREATE SCHEMA 模式名 AUTHORIZATION 用户名
  CREATE TABLE ...
  CREATE VIEW ...
  CREATE DOMAIN ...;
```

删除模式：

```sql
DROP SCHEMA 模式名 CASCADE;
DROP SCHEMA 模式名 RESTRICT;
```

- `CASCADE`：级联删除，模式内对象也一起删。
- `RESTRICT`：受限删除，只有模式为空时才允许删。

### 2.2 创建表 `CREATE TABLE`

**题型口令：**“设计/定义学生表、课程表、选课表，并带主键、外键、取值约束”。

```sql
CREATE TABLE 表名
(
  列名 数据类型 [列级完整性约束],
  列名 数据类型 [列级完整性约束],
  [表级完整性约束],
  ...
);
```

常见数据类型：

| 类型 | 用法 |
| --- | --- |
| `CHAR(n)` | 定长字符串，如学号、课程号 |
| `VARCHAR(n)` | 变长字符串，如姓名、院系 |
| `INT` / `SMALLINT` | 整数 |
| `REAL` / `DOUBLE` | 实数 |
| `DATE` | 日期 |
| `BOOLEAN` | 布尔值 |

完整性约束模板：

```sql
NOT NULL
UNIQUE
PRIMARY KEY (列名, ...)
FOREIGN KEY (本表列) REFERENCES 被参照表(列)
CHECK (条件)
DEFAULT 默认值
```

学生表模板：

```sql
CREATE TABLE Student
(
  sno   CHAR(5) PRIMARY KEY,
  sname CHAR(8) NOT NULL,
  sex   CHAR(2),
  age   SMALLINT,
  dept  CHAR(20),
  CHECK (sex IN ('男', '女'))
);
```

课程表模板：

```sql
CREATE TABLE Course
(
  cno    CHAR(4) PRIMARY KEY,
  cname  CHAR(10) NOT NULL,
  pcno   CHAR(4),
  credit SMALLINT,
  FOREIGN KEY (pcno) REFERENCES Course(cno)
);
```

选课表模板：

```sql
CREATE TABLE SC
(
  sno   CHAR(5),
  cno   CHAR(4),
  grade SMALLINT,
  PRIMARY KEY (sno, cno),
  FOREIGN KEY (sno) REFERENCES Student(sno),
  FOREIGN KEY (cno) REFERENCES Course(cno),
  CHECK (grade IS NULL OR grade BETWEEN 0 AND 100)
);
```

**易错点：**

- 主键自动满足唯一且非空。
- 外键列的数据类型和被参照列要一致或兼容。
- 多属性主键必须写成表级约束：`PRIMARY KEY (sno, cno)`。
- `CHECK` 是元组或列的取值约束，不是查询条件。

### 2.3 修改表 `ALTER TABLE`

新增列：

```sql
ALTER TABLE 表名
ADD 列名 数据类型 [完整性约束];
```

删除列：

```sql
ALTER TABLE 表名
DROP 列名 CASCADE;
ALTER TABLE 表名
DROP 列名 RESTRICT;
```

修改列类型：

```sql
ALTER TABLE 表名
MODIFY 列名 新数据类型;
```

新增/删除主键：

```sql
ALTER TABLE 表名
ADD PRIMARY KEY (列名, ...);

ALTER TABLE 表名
DROP PRIMARY KEY;
```

### 2.4 删除表 `DROP TABLE`

```sql
DROP TABLE 表名 CASCADE;
DROP TABLE 表名 RESTRICT;
```

- `CASCADE`：删除基本表，同时自动删除依赖它的视图、索引、触发器、约束等。
- `RESTRICT`：只有没有相关依赖对象时才允许删除。

### 2.5 域 `CREATE DOMAIN`

**题型口令：**“定义一个可复用的数据类型，并限定取值范围/默认值”。

```sql
CREATE DOMAIN 域名 AS 数据类型
  [DEFAULT 默认值]
  [CHECK (VALUE 条件)];
```

例：

```sql
CREATE DOMAIN item_id NUMERIC(4) DEFAULT 0
  CHECK (VALUE IS NOT NULL);
```

### 2.6 索引

**题型口令：**“为某列建立索引/唯一索引/聚簇索引，提高查询效率”。

```sql
CREATE [UNIQUE] [CLUSTER] INDEX 索引名
ON 表名 (列名 [ASC | DESC], 列名 [ASC | DESC], ...);
```

例：

```sql
CREATE UNIQUE INDEX Stno ON Student(sno);
CREATE UNIQUE INDEX Scno ON SC(sno, cno DESC);
```

删除索引：

```sql
DROP INDEX 索引名;
```

**易错点：**

- `UNIQUE` 要求索引列取值唯一。
- `CLUSTER` 表示索引顺序与表中记录物理顺序尽量一致。
- 索引一般不改变查询结果，只影响效率。

## 3. DML 查询：`SELECT` 解题模板

### 3.1 单表查询

#### 选择列

```sql
SELECT sname, sno, dept
FROM Student;
```

#### 表达式与别名

```sql
SELECT sname, 2026 - age AS birth_year
FROM Student;
```

#### 去重

```sql
SELECT DISTINCT sno
FROM SC
WHERE grade < 60;
```

#### 比较与范围

```sql
SELECT sno
FROM SC
WHERE cno = '004'
  AND grade BETWEEN 85 AND 95;
```

`BETWEEN a AND b` 等价于 `>= a AND <= b`。

#### 集合条件 `IN`

```sql
SELECT sname, sex
FROM Student
WHERE dept IN ('信息院', '数学系', '计算机学院');
```

否定：

```sql
WHERE dept NOT IN (...)
```

#### 字符匹配 `LIKE`

| 模板 | 含义 |
| --- | --- |
| `LIKE '王%'` | 以“王”开头 |
| `LIKE '%数据库%'` | 包含“数据库” |
| `LIKE '__阳%'` | 第 3 个字是“阳” |
| `LIKE '数据\_%' ESCAPE '\'` | 把 `_` 当普通字符 |

```sql
SELECT *
FROM Course
WHERE cname LIKE '数据\_%' ESCAPE '\';
```

#### 空值 `NULL`

```sql
WHERE grade IS NULL
WHERE grade IS NOT NULL
```

**不能写：**

```sql
WHERE grade = NULL
```

### 3.2 排序

```sql
SELECT sno, sname
FROM Student
WHERE sex = '男'
ORDER BY dept ASC, age DESC;
```

- `ASC`：升序，默认。
- `DESC`：降序。
- 多列排序时，先按第 1 列排，第 1 列相同再按第 2 列排。

### 3.3 聚合函数

| 函数 | 含义 |
| --- | --- |
| `COUNT(*)` | 统计元组个数 |
| `COUNT([DISTINCT] 列名)` | 统计某列非空值个数，可去重 |
| `SUM([DISTINCT] 列名)` | 求和 |
| `AVG([DISTINCT] 列名)` | 平均值 |
| `MAX(列名)` | 最大值 |
| `MIN(列名)` | 最小值 |

例：

```sql
SELECT COUNT(DISTINCT sno) AS student_count
FROM SC;

SELECT MAX(grade) AS max_grade,
       MIN(grade) AS min_grade,
       SUM(grade) AS total_grade
FROM SC
WHERE cno = '001' AND grade >= 60;
```

**易错点：**

- `COUNT(*)` 包括空值行。
- `COUNT(列名)` 不统计该列为 `NULL` 的行。
- 聚合函数通常不能直接写在 `WHERE`，要写在 `HAVING` 或子查询里。

### 3.4 分组查询 `GROUP BY ... HAVING`

**题型口令：**“每个学生/每门课/每个院系分别统计”。

```sql
SELECT 分组列, 聚合函数(列名)
FROM 表名
WHERE 分组前筛选条件
GROUP BY 分组列
HAVING 分组后筛选条件;
```

例：查询每个学生的平均成绩，只保留平均成绩大于 88 的学生。

```sql
SELECT sno, AVG(grade) AS avg_grade
FROM SC
GROUP BY sno
HAVING AVG(grade) > 88;
```

例：查询男学生中湖北籍且年龄小于 30 的每个年龄组人数，保留人数超过 30 的组。

```sql
SELECT age, COUNT(sno) AS number
FROM Student
WHERE sex = '男' AND place = '湖北'
GROUP BY age
HAVING COUNT(sno) > 30
ORDER BY number DESC;
```

**易错点：**

- `WHERE` 先于分组，筛“行”。
- `HAVING` 后于分组，筛“组”。
- `SELECT` 中的非聚合列，一般必须出现在 `GROUP BY` 中。

### 3.5 连接查询

#### 普通连接

**题型口令：**“查询学生情况及其选课成绩”，涉及 `Student` 与 `SC`。

```sql
SELECT Student.sno, sname, sex, age, dept, cno, grade
FROM Student, SC
WHERE Student.sno = SC.sno;
```

更推荐别名写法：

```sql
SELECT s.sno, s.sname, sc.cno, sc.grade
FROM Student AS s, SC AS sc
WHERE s.sno = sc.sno;
```

#### 多表连接

```sql
SELECT s.sname, c.cname, sc.grade
FROM Student AS s, SC AS sc, Course AS c
WHERE s.sno = sc.sno
  AND sc.cno = c.cno;
```

#### 自连接

**题型口令：**“找出年龄比王迎大的同学”。

```sql
SELECT s1.sname, s1.age
FROM Student AS s1, Student AS s2
WHERE s1.age > s2.age
  AND s2.sname = '王迎';
```

**自连接必须给同一张表起不同别名。**

#### 自然连接与外连接

```sql
SELECT *
FROM Student NATURAL JOIN SC;
```

外连接：

```sql
SELECT *
FROM Student LEFT OUTER JOIN SC
ON Student.sno = SC.sno;
```

| 连接 | 保留哪些不匹配元组 |
| --- | --- |
| `LEFT OUTER JOIN` | 左表 |
| `RIGHT OUTER JOIN` | 右表 |
| `FULL OUTER JOIN` | 左右两表 |

### 3.6 嵌套查询

#### `IN` 子查询

**题型口令：**“查询与某人同系的学生”。

```sql
SELECT sno, sname, dept
FROM Student
WHERE dept IN
(
  SELECT dept
  FROM Student
  WHERE sname = '刘晨'
);
```

#### 比较运算符子查询

**题型口令：**“年龄超过平均年龄”。

```sql
SELECT sname
FROM Student
WHERE age >
(
  SELECT AVG(age)
  FROM Student
);
```

#### `ANY` / `SOME` / `ALL`

| 模板 | 含义 |
| --- | --- |
| `x > ANY(子查询)` | x 大于子查询结果中的某一个值 |
| `x > ALL(子查询)` | x 大于子查询结果中的所有值 |
| `x < ANY(子查询)` | x 小于某一个值 |
| `x < ALL(子查询)` | x 小于所有值 |

例：查询比信息院某学生年龄小的学生。

```sql
SELECT sname, age
FROM Student
WHERE age < ANY
(
  SELECT age
  FROM Student
  WHERE dept = '信息院'
)
AND dept <> '信息院';
```

#### `EXISTS` 相关子查询

**题型口令：**“存在/至少有一条满足条件的记录”。

```sql
SELECT sname, sno
FROM Student
WHERE EXISTS
(
  SELECT *
  FROM SC
  WHERE SC.sno = Student.sno
    AND cno = '005'
);
```

#### `NOT EXISTS` 与“全部”题

SQL 没有直接的全称量词，遇到“全部/所有/每一门”通常改写为：

> 不存在一个对象，使得该学生没有满足它。

模板：

```sql
SELECT s.sno, s.sname
FROM Student AS s
WHERE NOT EXISTS
(
  SELECT *
  FROM Course AS c
  WHERE NOT EXISTS
  (
    SELECT *
    FROM SC AS sc
    WHERE sc.sno = s.sno
      AND sc.cno = c.cno
  )
);
```

例：查询选修了全部课程的学生姓名。

```sql
SELECT s.sname
FROM Student AS s
WHERE NOT EXISTS
(
  SELECT *
  FROM Course AS c
  WHERE NOT EXISTS
  (
    SELECT *
    FROM SC AS sc
    WHERE sc.sno = s.sno
      AND sc.cno = c.cno
  )
);
```

例：查询至少选修了学生 `200101` 所选全部课程的学生。

```sql
SELECT DISTINCT scx.sno
FROM SC AS scx
WHERE NOT EXISTS
(
  SELECT *
  FROM SC AS scy
  WHERE scy.sno = '200101'
    AND NOT EXISTS
    (
      SELECT *
      FROM SC AS scz
      WHERE scz.sno = scx.sno
        AND scz.cno = scy.cno
    )
);
```

### 3.7 集合运算

| 运算 | 含义 |
| --- | --- |
| `UNION` | 并 |
| `INTERSECT` | 交 |
| `EXCEPT` | 差 |

例：选修了 `001` 或 `002`，但没有选 `003` 的学生。

```sql
(
  SELECT sno
  FROM SC
  WHERE cno = '001' OR cno = '002'
)
EXCEPT
(
  SELECT sno
  FROM SC
  WHERE cno = '003'
);
```

## 4. DML 更新：插入、修改、删除

### 4.1 插入单行

```sql
INSERT INTO 表名 [(列名1, 列名2, ...)]
VALUES (常量1, 常量2, ...);
```

例：

```sql
INSERT INTO Student
VALUES ('200510', '王雪', '女', 20, '英语系');
```

**易错点：**

- 省略列名时，`VALUES` 必须与表中列的顺序、个数、类型一致。
- 指定列名时，未给值的列取默认值或 `NULL`。
- `NOT NULL` 列不能插入空值。

### 4.2 插入查询结果

```sql
INSERT INTO 目标表 [(列名, ...)]
SELECT ...
FROM ...
WHERE ...
GROUP BY ...
HAVING ...;
```

例：把平均成绩大于等于 90 的学生插入统计表。

```sql
INSERT INTO st1_grade(sno, name, avg)
SELECT Student.sno, sname, AVG(grade) AS avg_good
FROM Student, SC
WHERE Student.sno = SC.sno
GROUP BY Student.sno, sname
HAVING AVG(grade) >= 90;
```

### 4.3 修改数据 `UPDATE`

```sql
UPDATE 表名
SET 列名 = 表达式, 列名 = 表达式, ...
WHERE 条件;
```

例：学生“左筱”在 `001` 课程考试中作弊，该课成绩作零分。

```sql
UPDATE SC
SET grade = 0
WHERE cno = '001'
  AND sno =
  (
    SELECT sno
    FROM Student
    WHERE sname = '左筱'
  );
```

例：低于课程平均成绩的 `008` 课程成绩提高 5%。

```sql
UPDATE SC
SET grade = grade * 1.05
WHERE cno = '008'
  AND grade <
  (
    SELECT AVG(grade)
    FROM SC
    WHERE cno = '008'
  );
```

**易错点：**没有 `WHERE` 会更新整张表。

### 4.4 删除数据 `DELETE`

```sql
DELETE
FROM 表名
WHERE 条件;
```

例：删除选课人数不足 10 人的课程记录。

```sql
DELETE
FROM Course
WHERE cno IN
(
  SELECT cno
  FROM SC
  GROUP BY cno
  HAVING COUNT(*) < 10
);
```

**易错点：**没有 `WHERE` 会删除表中全部元组，但表结构仍存在。

## 5. 更多数据库对象：视图

### 5.1 视图概念

- 视图是从一个或多个基本表或视图导出的虚表。
- 数据库只保存视图定义，通常不保存视图对应的数据。
- 对视图的查询会转换为对基本表的查询，称为视图消解。

视图作用：

- 简化复杂查询。
- 多角度组织数据。
- 提高数据安全性，只暴露必要列或行。
- 提高逻辑独立性。

### 5.2 创建视图

```sql
CREATE VIEW 视图名 [(列名, 列名, ...)]
AS 子查询
[WITH CHECK OPTION];
```

例：建立信息院学生视图。

```sql
CREATE VIEW D_Student AS
SELECT sno, sname, age
FROM Student
WHERE dept = '信息院';
```

带检查选项：

```sql
CREATE VIEW D_Student AS
SELECT sno, sname, age
FROM Student
WHERE dept = '信息院'
WITH CHECK OPTION;
```

`WITH CHECK OPTION`：通过视图插入或更新的数据，必须仍满足视图定义中的 `WHERE` 条件。

### 5.3 行列子集视图

```sql
CREATE VIEW IS_S1(sno, sname, grade) AS
SELECT Student.sno, sname, grade
FROM Student, SC
WHERE Student.sno = SC.sno
  AND dept = '信息系'
  AND cno = '1';
```

### 5.4 视图上的视图

```sql
CREATE VIEW IS_S2 AS
SELECT sno, sname, grade
FROM IS_S1
WHERE grade >= 90;
```

### 5.5 带虚拟列的视图

**题型口令：**“视图中含计算列，如出生年份”。

```sql
CREATE VIEW BT_S(sno, sname, sbirth) AS
SELECT sno, sname, 2026 - age
FROM Student;
```

### 5.6 分组视图

```sql
CREATE VIEW S_G(sno, gavg) AS
SELECT sno, AVG(grade)
FROM SC
GROUP BY sno;
```

### 5.7 删除视图

```sql
DROP VIEW 视图名 CASCADE;
DROP VIEW 视图名 RESTRICT;
```

- `CASCADE`：删除该视图及依赖它的其他视图。
- `RESTRICT`：只有没有依赖对象时才允许删除。

### 5.8 更新视图的限制

一般不能更新的视图：

- 由两个或两个以上基本表导出。
- 视图列来自表达式或聚合函数。
- 定义中含 `GROUP BY`。
- 定义中含 `DISTINCT`。
- 定义中嵌套查询，并且内层查询的 `FROM` 涉及导出该视图的基本表。
- 一个不允许更新的视图上再定义的视图，也通常不允许更新。

### 5.9 物化视图

```sql
CREATE MATERIALIZED VIEW ...
```

- 普通视图：只保存定义，查询时再计算。
- 物化视图：保存查询结果数据，读取快，但基本表变化后需要同步维护。

## 6. DCL：授权、回收、角色

### 6.1 授权 `GRANT`

```sql
GRANT 权限 [, 权限 ...]
ON 对象类型 对象名
TO 用户 [, 用户 ...]
[WITH GRANT OPTION];
```

常见权限：`SELECT`，`INSERT`，`UPDATE`，`DELETE`，`ALL`。

例：把对表 `SC` 的查询和修改学号的权限授给所有用户。

```sql
GRANT SELECT, UPDATE(sno)
ON TABLE SC
TO PUBLIC;
```

例：把对表 `SC` 的插入权限授给用户 `U5`，并允许其继续转授。

```sql
GRANT INSERT
ON TABLE SC
TO U5
WITH GRANT OPTION;
```

**易错点：**

- `PUBLIC` 表示所有用户。
- `WITH GRANT OPTION` 表示被授权者还能把该权限继续授给别人。

### 6.2 回收权限 `REVOKE`

```sql
REVOKE 权限 [, 权限 ...]
ON 对象类型 对象名
FROM 用户 [, 用户 ...]
[CASCADE | RESTRICT];
```

例：

```sql
REVOKE INSERT
ON TABLE SC
FROM U5 CASCADE;
```

- `CASCADE`：级联收回由该用户继续转授出去的权限。
- `RESTRICT`：若存在转授权限，拒绝回收。

### 6.3 角色

创建角色：

```sql
CREATE ROLE 角色名;
```

给角色授权：

```sql
GRANT SELECT, INSERT
ON TABLE Student
TO C1;
```

把角色授给用户：

```sql
GRANT C1
TO U1, U2, U3;
```

给角色增加权限：

```sql
GRANT UPDATE, DELETE
ON TABLE Student
TO C1;
```

从角色回收权限：

```sql
REVOKE INSERT
ON TABLE Student
FROM C1;
```

## 7. 高频题型套用模板

### 7.1 “查询选修某课程的学生姓名”

```sql
SELECT s.sname
FROM Student AS s, SC AS sc
WHERE s.sno = sc.sno
  AND sc.cno = '课程号';
```

### 7.2 “查询没有选修某课程的学生”

```sql
SELECT s.sno, s.sname
FROM Student AS s
WHERE NOT EXISTS
(
  SELECT *
  FROM SC AS sc
  WHERE sc.sno = s.sno
    AND sc.cno = '课程号'
);
```

### 7.3 “查询选修了某学生所选全部课程的学生”

```sql
SELECT s.sno, s.sname
FROM Student AS s
WHERE NOT EXISTS
(
  SELECT *
  FROM SC AS sc1
  WHERE sc1.sno = '指定学生学号'
    AND NOT EXISTS
    (
      SELECT *
      FROM SC AS sc2
      WHERE sc2.sno = s.sno
        AND sc2.cno = sc1.cno
    )
);
```

### 7.4 “查询每门课的选课人数和平均成绩”

```sql
SELECT cno, COUNT(*) AS student_count, AVG(grade) AS avg_grade
FROM SC
GROUP BY cno;
```

### 7.5 “查询平均成绩最高的学生”

写法一：`ALL`。

```sql
SELECT sno
FROM SC
GROUP BY sno
HAVING AVG(grade) >= ALL
(
  SELECT AVG(grade)
  FROM SC
  GROUP BY sno
);
```

写法二：嵌套求最大平均值。

```sql
SELECT sno
FROM SC
GROUP BY sno
HAVING AVG(grade) =
(
  SELECT MAX(avg_grade)
  FROM
  (
    SELECT AVG(grade) AS avg_grade
    FROM SC
    GROUP BY sno
  ) AS T
);
```

### 7.6 “查询成绩不及格学生学号”

```sql
SELECT DISTINCT sno
FROM SC
WHERE grade < 60;
```

### 7.7 “查询某系学生平均年龄”

```sql
SELECT AVG(age) AS avg_age
FROM Student
WHERE dept = '系名';
```

### 7.8 “查询每个院系人数”

```sql
SELECT dept, COUNT(*) AS student_count
FROM Student
GROUP BY dept;
```

### 7.9 “删除没有人选修的课程”

```sql
DELETE
FROM Course
WHERE NOT EXISTS
(
  SELECT *
  FROM SC
  WHERE SC.cno = Course.cno
);
```

### 7.10 “建立只包含某系学生的视图”

```sql
CREATE VIEW V_Dept_Student AS
SELECT sno, sname, age
FROM Student
WHERE dept = '系名'
WITH CHECK OPTION;
```

## 8. 考场易错清单

- `WHERE` 不能直接写聚合条件，聚合条件用 `HAVING`。
- `NULL` 只能用 `IS NULL` / `IS NOT NULL` 判断。
- 多表查询必须写连接条件，否则会得到笛卡尔积。
- 自连接必须给同一张表起不同别名。
- “全部/所有/每一门”优先想到双重 `NOT EXISTS`。
- `NOT IN` 遇到子查询中含 `NULL` 时容易出错，考试中更稳的是 `NOT EXISTS`。
- `DISTINCT` 作用于整行选择结果，不是只作用于某一个显示列。
- `UPDATE` 和 `DELETE` 先写 `WHERE` 再检查，否则可能影响全表。
- 视图不一定可更新，尤其是含连接、分组、聚合、表达式、`DISTINCT` 的视图。
- `CASCADE` 是级联处理，`RESTRICT` 是有依赖就拒绝。
