# 第04章 规范化理论：开卷考试解题思路与模板

> 适用范围：函数依赖、范式、规范化、Armstrong 公理、属性闭包、候选键、最小函数依赖集、无损连接分解、保持函数依赖分解、3NF/BCNF 分解算法。

## 0. 考场快速判题地图

| 题目关键词 | 题型 | 先用什么模板 |
| --- | --- | --- |
| 判断某个 `X -> Y` 是否成立 | 函数依赖判断 | 看“同 X 是否必同 Y” |
| 判断 `F` 能否推出 `X -> Y` | 逻辑蕴涵 | 求 `X_F+`，看是否包含 `Y` |
| 求候选键/码 | 键计算 | 属性分类 `L/R/LR/NLR` + 闭包 |
| 判断 1NF/2NF/3NF/BCNF | 范式判定 | 先求候选键和主属性 |
| 求最小函数依赖集 | 最小覆盖 | 拆右部、删左部多余属性、删冗余依赖 |
| 判断分解是否无损 | 分解判定 | 二元公式或 Chase 表格法 |
| 判断是否保持函数依赖 | 分解判定 | 求各子模式投影依赖，再看能否推出原依赖 |
| 分解到 3NF | 分解算法 | 最小依赖集 + 分组 + 补候选键 |
| 分解到 BCNF | 分解算法 | 找违反 BCNF 的依赖，反复二分 |

**规范化题通用顺序：**

1. 写出关系模式 `R(U, F)`，确认属性全集 `U` 和函数依赖集 `F`。
2. 求候选键：先找必须在码中的属性，再用闭包验证。
3. 划分主属性和非主属性。
4. 判范式：按 `1NF -> 2NF -> 3NF -> BCNF` 逐级检查。
5. 若要求分解：先判断目标是 3NF 还是 BCNF，再选对应算法。

## 1. 关系模式、异常与规范化目标

### 1.1 关系模式形式化定义

关系模式是五元组：

```text
R(U, D, DOM, F)
```

| 符号 | 含义 |
| --- | --- |
| `R` | 关系名 |
| `U` | 属性集合 |
| `D` | 属性来自的域 |
| `DOM` | 属性到域的映射 |
| `F` | 属性间数据依赖集合，考试中主要是函数依赖 |

通常解题时简写为：

```text
R(U, F)
```

### 1.2 不规范关系的典型问题

以课件中的学生选课关系为例：

```text
S(SNO, SNAME, CLASS, CNO, TNO, TNAME, TAGE, ADDRESS, GRADE)
```

可能存在的问题：

| 问题 | 表现 | 原因 |
| --- | --- | --- |
| 数据冗余 | 教师姓名、年龄、地址重复出现 | 一个教师可能教多门课、多个学生选同一课 |
| 更新异常 | 修改教师地址要改很多行 | 同一事实存储多次 |
| 插入异常 | 新教师暂时没开课时无法插入 | 依赖于选课记录才能保存教师信息 |
| 删除异常 | 删除最后一条选课记录会丢失课程或教师信息 | 多类实体混在一张表中 |

规范化目标：

- 减少冗余。
- 避免插入、删除、更新异常。
- 让关系模式满足更高范式。
- 分解后尽量满足无损连接和保持函数依赖。

课件示例分解思路：

```text
SN(SNO, SNAME, CLASS)
CT(CNO, TNO)
TA(TNO, TNAME, TAGE, ADDRESS)
SC(SNO, CNO, GRADE)
```

## 2. 函数依赖基础

### 2.1 函数依赖定义

设 `R(U)` 是关系模式，`X`、`Y` 是 `U` 的子集。若对 `R` 的任意合法关系实例 `r`，任意两个元组 `t1`、`t2`：

```text
t1[X] = t2[X]  =>  t1[Y] = t2[Y]
```

则称 `X` 函数决定 `Y`，记作：

```text
X -> Y
```

解题口令：

> 只要两个元组的 X 值相同，Y 值就必然相同，则 `X -> Y` 成立。

### 2.2 属性间关系与函数依赖

| 属性关系 | 函数依赖情况 | 例子 |
| --- | --- | --- |
| 1:1 | `X -> Y` 且 `Y -> X` | 身份证号与一个人的唯一编号 |
| m:1 | `X -> Y`，但 `Y -/> X` | 学号决定姓名，姓名不一定决定学号 |
| 1:m | `Y -> X`，但 `X -/> Y` | 一个班级对应多个学生 |
| m:n | 通常无函数依赖 | 学生和课程 |

### 2.3 平凡、非平凡、完全、部分、传递依赖

| 类型 | 判定 | 记忆 |
| --- | --- | --- |
| 平凡函数依赖 | `Y ⊆ X` 时 `X -> Y` | 自己决定自己的一部分 |
| 非平凡函数依赖 | `Y ⊄ X` 时 `X -> Y` | 有实质信息 |
| 完全函数依赖 | `X -> Y`，且 `X` 的任一真子集都不能决定 `Y` | 左边少一个都不行 |
| 部分函数依赖 | `X -> Y`，但存在真子集 `X' -> Y` | 左边有多余属性 |
| 传递函数依赖 | `X -> Y`，`Y -> Z`，且 `Y -/> X`，则 `X -> Z` | 中间绕了一层 |

常见写法：

```text
X -> Y          函数依赖
X -/> Y         不能函数决定
X ->f Y         完全函数依赖
X ->p Y         部分函数依赖
X ->t Y         传递函数依赖
```

## 3. 键、候选键、主属性

### 3.1 基本概念

设 `R(U, F)`，`K ⊆ U`。

| 概念 | 判定 |
| --- | --- |
| 超键 | `K_F+ = U` |
| 候选键 | `K_F+ = U`，且 `K` 的任一真子集闭包都不是 `U` |
| 主键 | 从候选键中选一个作为标识 |
| 主属性 | 出现在任一候选键中的属性 |
| 非主属性 | 不出现在任何候选键中的属性 |

**候选键判定一句话：**

> 能推出全集，并且不能再删属性。

### 3.2 属性分类法求候选键

给定 `F`，把属性分为四类：

| 类别 | 定义 | 对候选键的影响 |
| --- | --- | --- |
| `L` | 只出现在依赖左部的属性 | 必在任一候选键中 |
| `R` | 只出现在依赖右部的属性 | 不在任一候选键中 |
| `LR` | 左部、右部都出现的属性 | 可能在候选键中 |
| `NLR` | 左部、右部都不出现的属性 | 必在任一候选键中 |

求候选键模板：

```text
1. 找 L、R、LR、NLR。
2. 必选属性 = L ∪ NLR。
3. 求 (L ∪ NLR)_F+。
4. 若闭包等于 U，则 L ∪ NLR 是唯一候选键。
5. 若闭包不等于 U，从 LR 中逐个/组合加入属性，求闭包。
6. 能推出 U 且最小的属性组，就是候选键。
```

**检查最小性：**

若某个候选键候选集为 `K`，必须检查每个 `A ∈ K`：

```text
(K - A)_F+ != U
```

否则 `A` 多余，`K` 不是候选键。

## 4. 范式判定模板

### 4.1 1NF

定义：

> 关系中每个属性值都是不可再分的原子值，则属于 1NF。

考场口令：

- 有重复组、数组、集合、嵌套表，不是 1NF。
- 只要是普通二维表，通常默认满足 1NF。

### 4.2 2NF

定义：

> 若 `R ∈ 1NF`，且每个非主属性都完全函数依赖于任一候选键，则 `R ∈ 2NF`。

判定口令：

```text
是否存在：候选键的真子集 -> 非主属性
若存在，则不是 2NF。
```

常见触发条件：

- 候选键是组合属性，如 `(SNO, CNO)`。
- 存在 `SNO -> SNAME`，`SNO -> CLASS` 这类只依赖组合键一部分的依赖。

结论：

- 若所有候选键都是单属性，通常一定满足 2NF。
- 2NF 消除非主属性对候选键的部分函数依赖。

分解到 2NF 模板：

```text
1. 找候选键 K。
2. 找出 K 的真子集 X -> 非主属性 A。
3. 分出 R1(X, A...)。
4. 原关系保留 K 和其余直接依赖于 K 的属性。
5. 对每个部分依赖重复上述过程。
```

### 4.3 3NF

定义一：

> 若 `R ∈ 2NF`，且每个非主属性都不传递依赖于任一候选键，则 `R ∈ 3NF`。

等价判定：

对 `F+` 中每一个非平凡函数依赖 `X -> A`，若满足以下任一条件，则符合 3NF：

```text
1. X 是超键；
2. A 是主属性。
```

判定口令：

```text
是否存在：候选键 K -> 中间属性 Y -> 非主属性 A
若存在，则不是 3NF。
```

例：

```text
(SNO, CNO) -> TNO
TNO -> TNAME
所以 (SNO, CNO) -> TNAME 是传递依赖
```

分解到 3NF 思路：

```text
把 X -> A 中的 X 和 A 放到一个子关系中，
让每个非主属性直接依赖于码，而不是依赖于其他非主属性。
```

### 4.4 BCNF

定义：

> 若 `R ∈ 1NF`，且 `F+` 中每个非平凡函数依赖 `X -> Y` 的左部 `X` 都包含 `R` 的某个候选键，则 `R ∈ BCNF`。

等价口令：

```text
每一个决定因素都必须是超键。
```

判定模板：

```text
对每个非平凡依赖 X -> A：
1. 求 X_F+。
2. 若 X_F+ = U，则 X 是超键，该依赖不违反 BCNF。
3. 若 X_F+ != U，则 X 不是超键，该依赖违反 BCNF。
```

关系：

```text
BCNF => 3NF => 2NF => 1NF
```

**区别 3NF 与 BCNF：**

- 3NF 允许 `X` 不是超键，但右部 `A` 是主属性。
- BCNF 不看右部是不是主属性，只要求左部 `X` 必须是超键。

### 4.5 4NF 简记

课件范式层次中出现 4NF。常用记法：

> 4NF 进一步消除非平凡且非函数依赖的多值依赖。

若题目只考函数依赖，通常重点到 BCNF；若出现多值依赖 `X ->-> Y`，再考虑 4NF。

## 5. Armstrong 公理与推理规则

### 5.1 Armstrong 三条基本公理

设 `U` 为属性全集，`X,Y,Z ⊆ U`。

| 公理 | 形式 | 说明 |
| --- | --- | --- |
| 自反律 | 若 `Y ⊆ X`，则 `X -> Y` | 平凡依赖 |
| 增广律 | 若 `X -> Y`，则 `XZ -> YZ` | 两边同时加属性 |
| 传递律 | 若 `X -> Y`，`Y -> Z`，则 `X -> Z` | 链式推出 |

### 5.2 常用推理规则

| 规则 | 形式 |
| --- | --- |
| 合并规则 | 若 `X -> Y`，`X -> Z`，则 `X -> YZ` |
| 分解规则 | 若 `X -> YZ`，则 `X -> Y`，`X -> Z` |
| 伪传递规则 | 若 `X -> Y`，`WY -> Z`，则 `WX -> Z` |

常用定理：

```text
X -> A1A2...Ak 成立
当且仅当
X -> A1, X -> A2, ..., X -> Ak 都成立
```

## 6. 闭包计算

### 6.1 函数依赖集闭包 `F+`

定义：

> 由 `F` 通过 Armstrong 公理能够推出的全部函数依赖集合，称为 `F` 的闭包，记作 `F+`。

直接求 `F+` 通常太大，考试一般改为求属性闭包。

### 6.2 属性闭包 `X_F+`

定义：

> 在函数依赖集 `F` 下，由属性集 `X` 能推出的全部属性集合，称为 `X` 关于 `F` 的闭包，记作 `X_F+`。

算法模板：

```text
输入：属性集 X，函数依赖集 F
输出：X_F+

1. 令 X+ = X。
2. 扫描 F 中每个依赖 Y -> Z。
3. 若 Y ⊆ X+，则令 X+ = X+ ∪ Z。
4. 重复第 2、3 步，直到 X+ 不再变化。
5. 输出 X+。
```

### 6.3 用闭包判断依赖是否可推出

题型：

```text
判断 F 是否蕴涵 X -> Y。
```

模板：

```text
1. 求 X_F+。
2. 若 Y ⊆ X_F+，则 F |= X -> Y。
3. 若 Y ⊄ X_F+，则 F 不能推出 X -> Y。
```

### 6.4 用闭包判断超键/候选键

```text
1. 求 X_F+。
2. 若 X_F+ = U，则 X 是超键。
3. 再检查 X 的任一真子集闭包是否都不是 U。
4. 若都不是 U，则 X 是候选键。
```

## 7. 最小函数依赖集

### 7.1 等价与覆盖

若 `F+ = G+`，则称 `F` 与 `G` 等价，也称 `F` 覆盖 `G` 且 `G` 覆盖 `F`。

判断 `F` 与 `G` 等价模板：

```text
1. 对 F 中每个 X -> Y，求 X_G+，若 Y ⊆ X_G+，则 G 能推出该依赖。
2. 对 G 中每个 X -> Y，求 X_F+，若 Y ⊆ X_F+，则 F 能推出该依赖。
3. 两边都能推出，则 F+ = G+。
```

### 7.2 最小函数依赖集定义

`Fm` 是 `F` 的最小函数依赖集，需要满足：

```text
1. 每个依赖右部只有单个属性。
2. 每个依赖左部没有多余属性。
3. 没有多余的函数依赖。
```

### 7.3 求最小函数依赖集算法

**第一步：拆右部。**

```text
X -> A1A2...Ak
拆成
X -> A1, X -> A2, ..., X -> Ak
```

**第二步：删除左部多余属性。**

对每个依赖：

```text
XY -> A
```

检查 `Y` 是否多余：

```text
1. 在当前 F 中求 X_F+。
2. 若 A ∈ X_F+，则 Y 是多余属性，改为 X -> A。
3. 若 A ∉ X_F+，则 Y 不能删除。
```

更稳的写法：

```text
对左部每个属性 B：
1. 令 X' = X - B。
2. 求 X'_F+。
3. 若右部 A ∈ X'_F+，则 B 多余。
```

**第三步：删除冗余函数依赖。**

对每个依赖：

```text
X -> A
```

检查它是否多余：

```text
1. 临时令 F' = F - {X -> A}。
2. 求 X_F'+。
3. 若 A ∈ X_F'+，则 X -> A 是冗余依赖，可以删除。
4. 否则保留。
```

**最终：**

```text
得到 Fm，且 Fm 与 F 等价。
```

### 7.4 最小依赖集易错点

- 必须先把右部拆成单属性。
- 删除左部属性时，一次只删一个，再重新检查。
- 删除某个依赖时，闭包必须在 `F - {该依赖}` 下求。
- 最小依赖集不一定唯一，但必须与原依赖集等价。

## 8. 分解：无损连接与保持函数依赖

### 8.1 分解的两个目标

设关系模式 `R` 分解为：

```text
ρ = {R1, R2, ..., Rk}
```

希望满足：

| 性质 | 含义 |
| --- | --- |
| 无损连接性 | 分解后自然连接不会丢失信息，也不会产生多余信息 |
| 保持函数依赖 | 原来的函数依赖可以在各子关系中检查，不必连接回原表 |

### 8.2 二元分解无损连接判定

若 `R` 分解为 `R1` 和 `R2`，则无损连接当且仅当满足下列任一条件：

```text
(R1 ∩ R2) -> (R1 - R2)
```

或

```text
(R1 ∩ R2) -> (R2 - R1)
```

其中依赖要在 `F+` 中成立。

考场模板：

```text
1. 求公共属性 C = R1 ∩ R2。
2. 求 C_F+。
3. 若 R1 ⊆ C_F+ 或 R2 ⊆ C_F+，则无损连接。
4. 否则不是无损连接。
```

### 8.3 多表分解无损连接 Chase 表格法

输入：

```text
R(A1, A2, ..., An)
F
ρ = {R1, R2, ..., Rk}
```

表格构造：

```text
1. 构造 k 行 n 列的表。
2. 第 i 行对应 Ri，第 j 列对应属性 Aj。
3. 若 Aj ∈ Ri，则填 aj。
4. 若 Aj ∉ Ri，则填 bij。
```

反复应用函数依赖：

```text
对每个 X -> Y：
1. 找出在 X 各列上符号完全相同的行。
2. 把这些行在 Y 列上的符号统一。
3. 如果其中有 aj，则全部改为 aj。
4. 如果没有 aj，则全部改为某个 b 符号，通常取行号最小的那个。
5. 重复直到表格不再变化。
```

判定：

```text
若某一行变成 a1, a2, ..., an，则分解具有无损连接性。
否则不具有无损连接性。
```

### 8.4 保持函数依赖判定

定义投影依赖：

```text
Fi = πRi(F)
```

含义：

> `Fi` 是只涉及 `Ri` 中属性、且能由 `F+` 推出的函数依赖集合。

判定模板：

```text
1. 对每个子模式 Ri，求 Fi = πRi(F)。
2. 令 G = F1 ∪ F2 ∪ ... ∪ Fk。
3. 对原 F 中每个依赖 X -> Y，求 X_G+。
4. 若所有 Y ⊆ X_G+，则保持函数依赖。
5. 若存在某个依赖推不出，则不保持函数依赖。
```

快判：

- 若原依赖 `X -> A` 的所有属性 `XA` 完全落在某个子模式中，通常容易保持。
- 若原依赖被拆散到多个子模式中，需要用闭包验证。

## 9. 3NF 分解算法

目标：

```text
把 R<U,F> 分解为 3NF，
并尽量保证无损连接和保持函数依赖。
```

课件算法模板：

```text
1. 最小化：求 F 的最小函数依赖集 Fm。

2. 排除：
   若 Fm 中有依赖 X -> A，且 XA = U，
   则直接输出 ρ = {R}，结束。

3. 独立属性：
   若 R 中某些属性未出现在 Fm 的任一依赖左部或右部，
   则把这些属性单独组成一个关系子模式。

4. 分组：
   对 Fm 中每个 X -> A，构造关系子模式 XA。
   若有 X -> A1, X -> A2, ..., X -> An，
   可合并为 X -> A1A2...An，
   构造子模式 X A1 A2 ... An。

5. 补候选键：
   若当前分解中没有任何一个子模式包含 R 的候选键，
   则添加一个由候选键属性构成的子模式。

6. 删除冗余子模式：
   若 Ri ⊆ Rj，则删除 Ri。
```

考试书写格式：

```text
先求 Fm = {...}
由 X1 -> A1A2 得 R1(X1,A1,A2)
由 X2 -> B 得 R2(X2,B)
...
检查是否包含候选键 K
若没有，补 Rk(K)
删除被包含的子模式
最终 ρ = {...}
```

**性质：**

- 通常保持函数依赖。
- 补入候选键后可保证无损连接。
- 不一定达到 BCNF。

## 10. BCNF 分解算法

目标：

```text
把 R<U,F> 分解为 BCNF，并保证无损连接。
```

算法模板：

```text
1. 令 ρ = {R}。
2. 检查 ρ 中每个关系模式 Ri 是否属于 BCNF。
3. 若某个 Ri 不属于 BCNF，
   则存在非平凡依赖 X -> A，
   其中 A ∉ X，且 X 不是 Ri 的超键。
4. 用 X -> A 分解 Ri：
   S1 = X A
   S2 = Ui - A
   用 S1、S2 替换 Ri。
5. 对新得到的子模式重复第 2 步。
6. 所有子模式都是 BCNF 时，输出 ρ。
```

更一般的右部多属性写法：

```text
若违反依赖为 X -> Y：
S1 = X ∪ Y
S2 = Ri - (Y - X)
```

**性质：**

- BCNF 分解保证无损连接。
- BCNF 分解不一定保持函数依赖。
- 每一步选哪个违反依赖，可能得到不同的 BCNF 分解。

## 11. 高频题型模板

### 11.1 判断一个依赖是否成立

题型：

```text
给出关系实例表，判断 X -> Y 是否成立。
```

步骤：

```text
1. 找出 X 列相同的所有元组对。
2. 检查这些元组的 Y 列是否也相同。
3. 只要存在 X 相同但 Y 不同，则 X -> Y 不成立。
4. 若所有 X 相同的元组 Y 都相同，则 X -> Y 在该实例上成立。
```

注意：

- 单个实例上成立，不代表在关系模式语义上一定成立。
- 单个实例上不成立，可以说明该函数依赖一定不成立。

### 11.2 求 `X_F+`

格式：

```text
X+ = {初始属性}
根据 A -> B，加入 ...
根据 C -> D，加入 ...
最终 X+ = {...}
```

判定句：

```text
因为 Y ⊆ X+，所以 X -> Y 成立。
因为 U ⊆ X+，所以 X 是超键。
```

### 11.3 求所有候选键

步骤：

```text
1. 划分 L/R/LR/NLR。
2. 写出必选属性 M = L ∪ NLR。
3. 求 M+。
4. 若 M+ = U，M 为唯一候选键。
5. 若 M+ != U，从 LR 中加入一个属性再求闭包。
6. 若单个不行，试 LR 中两个属性组合。
7. 每得到一个能推出 U 的集合，都检查最小性。
8. 列出全部最小超键，即候选键。
```

答题句式：

```text
L = {...}, R = {...}, LR = {...}, NLR = {...}
M = L ∪ NLR = {...}
M+ = {...}
所以候选键为 {...}
主属性为 {...}
非主属性为 {...}
```

### 11.4 判断最高范式

步骤：

```text
1. 默认先检查 1NF。
2. 求候选键。
3. 得出主属性和非主属性。
4. 查是否有非主属性对候选键的部分依赖：
   有则最高为 1NF。
5. 查是否有非主属性对候选键的传递依赖：
   有则最高为 2NF。
6. 对每个非平凡依赖 X -> A：
   若 X 不是超键且 A 不是主属性，则不是 3NF。
7. 对每个非平凡依赖 X -> A：
   若 X 不是超键，则不是 BCNF。
```

常用结论句：

```text
存在候选键真子集 X -> 非主属性 A，所以不满足 2NF，最高为 1NF。
```

```text
存在 K -> Y，Y -> A，其中 A 为非主属性，所以不满足 3NF，最高为 2NF。
```

```text
对依赖 X -> A，X 不是超键，但 A 是主属性，因此满足 3NF，不满足 BCNF。
```

```text
所有非平凡依赖左部都是超键，因此满足 BCNF。
```

### 11.5 判断两个依赖集是否等价

模板：

```text
要证 F 与 G 等价：

1. 证 F 覆盖 G：
   对 G 中每个 X -> Y，求 X_F+，若 Y ⊆ X_F+，成立。

2. 证 G 覆盖 F：
   对 F 中每个 X -> Y，求 X_G+，若 Y ⊆ X_G+，成立。

3. 两边都成立，所以 F+ = G+。
```

### 11.6 求最小函数依赖集

答题格式：

```text
原 F = {...}

第一步，拆右部：
F1 = {...}

第二步，删左部多余属性：
检查 AB -> C：
求 A+ = {...}
若 C ∈ A+，则 B 多余，AB -> C 改为 A -> C。
...
得到 F2 = {...}

第三步，删冗余依赖：
检查 X -> A：
在 F2 - {X -> A} 下求 X+ = {...}
若 A ∈ X+，删除 X -> A。
...
最终 Fm = {...}
```

### 11.7 判断二元分解是否无损

题型：

```text
R(A,B,C), F={A -> B, A -> C}, 分解 ρ={AB, AC}
```

模板：

```text
R1 ∩ R2 = A
A+ = ABC
因为 R1 ⊆ A+ 或 R2 ⊆ A+，
所以该分解具有无损连接性。
```

反例句式：

```text
R1 ∩ R2 = C
C+ = C
不能推出 R1 或 R2，
所以该分解不具有无损连接性。
```

### 11.8 判断是否保持函数依赖

模板：

```text
ρ = {R1, R2, ..., Rk}
F1 = πR1(F) = {...}
F2 = πR2(F) = {...}
...
G = F1 ∪ F2 ∪ ... ∪ Fk

对 F 中每个依赖 X -> Y：
求 X_G+。
若全部能推出，则保持函数依赖；
若某个推不出，则不保持函数依赖。
```

常见答题句：

```text
依赖 C -> B 在任何一个子模式中都无法检查，
且不能由投影依赖推出，
所以该分解不保持函数依赖。
```

### 11.9 分解到 3NF

模板：

```text
1. 求最小依赖集 Fm。
2. 合并同左部依赖。
3. 每个依赖 X -> A... 生成一个子关系 R(X,A,...)。
4. 检查是否有子关系包含候选键。
5. 若没有，补一个候选键关系。
6. 删除被其他子关系包含的子关系。
7. 输出 3NF 分解。
```

答题句：

```text
Fm = {C -> D, E -> A, B -> A, B -> C, CA -> E}
候选键为 BG。
由 Fm 得到 CD, EA, BA, BC, CAE。
当前没有子模式包含 BG，因此补 BG。
最终 ρ = {CD, EA, BA, BC, CAE, BG}。
```

### 11.10 分解到 BCNF

模板：

```text
1. 先求候选键，判断 R 是否 BCNF。
2. 找一个违反 BCNF 的依赖 X -> A。
3. 分解为：
   R1 = X A
   R2 = U - A
4. 分别判断 R1、R2 是否 BCNF。
5. 若仍有违反依赖，继续分解。
6. 全部子关系都满足 BCNF 后输出。
```

课件例题句式：

```text
设 R(C,T,H,R,S,G)，F={CS -> G, C -> T, TH -> R, HR -> C, HS -> R}
候选键为 HS，R 不是 BCNF。
选 CS -> G：
S1 = CSG，是 BCNF；
S2 = CTHRS，不是 BCNF。
在 S2 中选 C -> T：
S1 = CT，是 BCNF；
S2 = CHRS，不是 BCNF。
在 CHRS 中选 HR -> C：
S1 = HRC，是 BCNF；
S2 = HRS，是 BCNF。
最终 ρ = {CSG, CT, HRC, HRS}。
```

## 12. 易错清单

- 判断范式前必须先求候选键，否则主属性和非主属性容易判错。
- 2NF 只关注非主属性对候选键的部分依赖。
- 3NF 关注非主属性传递依赖，也可用“左部是超键或右部是主属性”检查。
- BCNF 要求所有非平凡依赖的左部都是超键，比 3NF 更严格。
- 组合候选键才容易出现部分依赖；单属性候选键一般不会违反 2NF。
- 求闭包时要重复扫描依赖集，直到没有新属性加入。
- 求最小依赖集时，右部必须先拆成单属性。
- 删除冗余依赖时，必须在去掉该依赖后的集合中求闭包。
- 二元分解无损只看公共属性能否决定某一边的差集。
- 多表无损连接用 Chase 表格法，有一行全变成 `a1...an` 才是无损。
- 3NF 分解通常能保持函数依赖；BCNF 分解通常无损，但可能不保持函数依赖。
- 不同的 BCNF 分解路径可能得到不同答案，关键是每个子模式都满足 BCNF 且分解无损。
