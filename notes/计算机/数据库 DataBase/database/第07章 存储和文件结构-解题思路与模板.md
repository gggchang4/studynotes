> 适用范围：存储介质、磁盘参数与调度、RAID、数据库逻辑/物理组织、段页式存储、块和记录地址、定长/变长记录、记录插删改、表组织、元数据组织、缓冲区、页面替换策略。

**本章答题通用口令：**

1. 只要涉及磁盘性能，先想“少 I/O、顺序读写、预取、缓冲”。
2. 只要涉及记录存储，先想“块是 I/O 基本单位，记录在块内组织”。
3. 只要涉及修改，先想“是否原地更新、是否有日志、是否需要写回磁盘”。
4. 只要涉及缓存，先想“命中率、替换策略、脏页、预写日志”。

## 1. 存储介质

### 1.1 存储介质分类

| 类型 | 例子 | 特点 |
| --- | --- | --- |
| 纸质/卡片/纸带 | 早期介质 | 低效，历史性 |
| 磁介质 | 磁带、磁盘、软盘、硬盘 HDD | 容量大，价格低，磁盘常用于数据库 |
| 光存储 | CD、DVD | 便宜，访问慢，多用于归档 |
| 电子介质 | 闪存 Flash、SSD、NVM | 随机访问快，价格较高 |

### 1.2 存储层次

从快到慢大致为：

```text
CPU 寄存器/缓存 -> DRAM 主存 -> 非易失性主存 -> SSD/闪存 -> HDD 磁盘 -> 光盘/磁带
```

一般规律：

| 方向 | 速度 | 单位价格 | 容量 | 持久性 |
| --- | --- | --- | --- | --- |
| 靠近 CPU | 快 | 贵 | 小 | 多数非持久 |
| 靠近磁盘/磁带 | 慢 | 便宜 | 大 | 持久 |

### 1.3 SSD 与 HDD 对比

| 维度 | SSD | HDD |
| --- | --- | --- |
| 随机存取 | 强 | 弱 |
| 顺序读写 | 很快 | 较快 |
| 寻道时间 | 极小，约 0.08ms-0.16ms | 明显，移动端约 4ms-15ms |
| 旋转延迟 | 无 | 有 |
| 碎片影响 | 无或较小 | 有 |
| 价格 | 较高 | 较低 |
| 容量 | 较小或较贵 | 大且便宜 |
| 寿命 | 写入次数有限 | 机械磨损 |

数据库优化口令：

```text
HDD：顺序读写远快于随机读写，优化重点是减少寻道和随机 I/O。
SSD：随机延迟较低，但仍要减少写放大和无谓 I/O。
```

## 2. 磁盘参数与访问时间

### 2.1 磁盘基本术语

| 术语 | 含义 |
| --- | --- |
| 盘面 | 磁盘表面 |
| 磁头 | 读写数据的部件 |
| 柱面 | 多个盘面上同半径磁道组成的集合 |
| 磁道 | 盘面上的同心圆轨道 |
| 扇区 | 磁道上的分段 |
| 块 | DBMS 或 OS 读写的基本单位 |
| 间隙 | 相邻块/扇区间的空隙 |

### 2.2 磁盘访问时间公式

磁盘访问一个块的时间通常为：

```text
T = Tseek + Trotate + Ttran
```

| 符号 | 含义 |
| --- | --- |
| `Tseek` | 寻道时间，磁头移动到目标柱面 |
| `Trotate` | 旋转延迟，等待目标块转到磁头下方 |
| `Ttran` | 传输时间，把块内容读入/写出 |

课件常用参数：

```text
Ttran = 数据块大小 / 传输速率
```

典型量级：

- HDD 寻道：约 4ms-15ms。
- HDD 旋转延迟：约 2ms-7ms。
- SSD 寻址：约 0.08ms-0.16ms。

### 2.3 磁盘访问计算模板

题型：

```text
给出请求柱面、到达时间、磁头当前位置，计算调度顺序和完成时间。
```

步骤：

```text
1. 写出每个请求：(柱面号, 到达时间)。
2. 确定初始磁头位置和扫描方向。
3. 每服务一个请求，计算：
   寻道时间 + 平均旋转延迟 + 传输时间。
4. 若请求尚未到达，不能提前服务。
5. 累加得到每个请求的完成时间。
```

常用答题句：

```text
由于磁盘 I/O 时间主要由寻道和旋转延迟构成，因此应尽量减少磁头移动距离，把随机读写转化为顺序读写。
```

## 3. 磁盘调度算法

### 3.1 调度目标

- 减少平均 I/O 等待时间。
- 减少磁头移动距离。
- 尽量把随机读写转化为顺序读写。
- 提高磁盘吞吐量。

### 3.2 先来先服务 FCFS

规则：

```text
按请求到达顺序处理。
```

优点：

- 简单公平。
- 不会饿死请求。

缺点：

- 可能导致磁头来回大幅移动，平均访问时间较长。

计算模板：

```text
当前柱面 = 初始柱面
按到达顺序依次处理请求：
  移动距离 = |目标柱面 - 当前柱面|
  本次时间 = 寻道时间(移动距离) + 旋转延迟 + 传输时间
  当前柱面 = 目标柱面
```

### 3.3 电梯算法 SCAN

规则：

```text
磁头沿一个方向移动，服务该方向上遇到的请求；
到达末端或无请求后，再反向移动。
```

优点：

- 减少来回摆动。
- 比 FCFS 更适合大量请求。

计算模板：

```text
1. 写出所有已经到达的请求。
2. 按当前扫描方向选择下一个柱面。
3. 沿方向依次处理请求。
4. 到端点或该方向无请求时换向。
5. 新到达的请求按其柱面位置插入等待队列。
```

### 3.4 磁盘性能优化技术

课件列出的优化技术：

```text
1. 磁盘调度算法
2. 磁盘缓冲区
3. 预取
4. 队列
5. 镜像磁盘
6. 软件缓存
```

记忆：

> 调度减少移动，缓冲减少 I/O，预取利用顺序性，镜像提高可靠性和读性能。

## 4. 存储故障与 RAID

### 4.1 存储故障分类

| 分类 | 含义 |
| --- | --- |
| 部分故障 | 部分数据损坏或部分设备失效 |
| 完全故障 | 整个设备或介质不可用 |
| 间歇故障 | 偶尔出错，可能恢复 |
| 永久故障 | 持续失效，必须修复或替换 |

检测方法：

- 校验和 `checksum`。
- 冗余校验。
- 读后验证。

### 4.2 常见故障处理思想

| 故障 | 处理 |
| --- | --- |
| 间歇性故障 | 读写后检查状态或校验和 |
| 稳定存储问题 | 稳定存储/镜像策略 |
| 错误处理 | 重试、回滚、使用日志 |
| 磁盘崩溃 | RAID、备份恢复 |

### 4.3 RAID 级别

| RAID | 关键思想 | 特点 |
| --- | --- | --- |
| RAID1 | 镜像 | 数据复制到冗余磁盘，读快、空间开销大 |
| RAID4 | 独立数据盘 + 固定校验盘 | 可恢复单盘故障，校验盘可能成为瓶颈 |
| RAID5 | 分布式校验 | 校验信息分布到多个磁盘，避免单校验盘瓶颈 |
| RAID6 | 双重校验 | 可容忍多盘失效，校验开销更高 |

### 4.4 RAID 校验计算模板

RAID4/5 常用按位异或校验：

```text
P = D1 XOR D2 XOR ... XOR Dn
```

若某个数据盘 `Di` 丢失：

```text
Di = P XOR D1 XOR ... XOR D(i-1) XOR D(i+1) XOR ... XOR Dn
```

答题句：

```text
由于异或满足 A XOR A = 0、A XOR 0 = A，因此任意单盘丢失时，可用其余数据盘和校验盘恢复。
```

### 4.5 RAID 易错点

- RAID 不是备份，不能代替长期备份和日志恢复。
- RAID1 主要靠镜像提高可靠性，容量利用率低。
- RAID4 的固定校验盘可能成为写瓶颈。
- RAID5 将校验分布到各盘。
- RAID6 能容忍更多磁盘故障，但写入代价更高。

## 5. 数据库的逻辑和物理组织

### 5.1 文件管理方式

课件给出两类方式：

```text
1. 一个数据库对应一个文件，由 OS 进行存储管理。
2. 一个 DB 对象对应一个或若干个文件，由 DBMS 进行存储管理。
```

现代 DBMS 通常自己管理存储，以便更精细控制数据布局和 I/O。

### 5.2 DBMS 页式存储层次

常见组织形式：

```text
数据库 -> 表空间 -> 段 -> 分区 -> 数据块
```

| 层次 | 含义 |
| --- | --- |
| 表空间 | 逻辑存储设备，可对应一个或多个物理文件 |
| 段 | 某类对象占用的空间，如数据段、索引段、回滚段、临时段 |
| 分区 | 一组连续的数据块 |
| 数据块 | 数据库最小 I/O 读写单位 |

课件强调：

```text
数据块是磁盘存取单元。
数据块大小通常为服务器 OS 块大小的整数倍。
```

### 5.3 逻辑结构与物理结构映射

| 逻辑结构 | 物理结构 |
| --- | --- |
| 表空间 | 文件 |
| 段 | 文件中的连续或非连续区域 |
| 分区 | 一组连续块 |
| 数据块 | 物理块 |
| 记录 | 块内偏移位置 |

简化记忆：

```text
文件 -> 块 -> 记录
```

物理上，文件被切成固定长度的存储单元；DBMS 以块为 I/O 基本单位。

## 6. 记录与块的组织

### 6.1 定长记录

定长记录由固定长度字段组成，记录长度固定。

记录通常包含：

```text
1. 记录头部信息
2. 字段值
3. 记录长度
4. 创建时间或时间戳
5. 指向模式的指针
```

优点：

- 定位简单。
- 第 `i` 条记录位置容易计算。
- 更新方便。

缺点：

- 可能浪费空间。
- 不适合可变长度字段或重复字段。

### 6.2 将定长记录装入块

块中通常存储：

```text
块头部 + 记录1 + 记录2 + ... + 空闲空间
```

块头部包含：

- 指向模式的信息。
- 与其他块的链接。
- 本块中记录目录。
- 块内可用空间。

同一块中的记录通常属于同一个关系。

### 6.3 块和记录地址

地址访问方式：

| 类型 | 含义 |
| --- | --- |
| 物理地址 | 直接描述设备和磁盘位置 |
| 逻辑地址 | 通过映射转换为物理地址 |

物理地址可能包含：

```text
主机名
设备 ID
柱面号
磁道号
块号
记录的块内偏移
```

常用记录地址：

```text
记录地址 = 块 ID + 块内偏移
```

逻辑地址转换：

```text
逻辑地址 -> 映射表 -> 物理地址
```

若移动记录，需要修改映射表，而不是所有引用它的位置。

### 6.4 块内间接访问

在块头部维护记录目录或偏移表：

```text
记录 ID -> 记录在块内的偏移
```

好处：

- 记录在块内移动时，只改目录。
- 外部引用可以保持稳定。
- 支持插入、删除和压缩。

### 6.5 指针 Swizzling

指针 swizzling 指把数据库地址转换为内存地址，以便在内存中快速访问。

混写指针策略：

| 策略 | 含义 |
| --- | --- |
| 自动混写 | 记录载入内存后自动把数据库地址换成内存指针 |
| 按需混写 | 访问到某个指针时再转换 |
| 无混写 | 始终使用数据库地址 |
| 程序控制混写 | 由程序显式控制转换 |

块写回时需要维护：

```text
dbAddr -> memAddr
memAddr -> dbAddr
```

即内存地址和数据库地址的双向映射。

### 6.6 固定与钉住的块

若块被其他混写指针引用，则该块在内存中被钉住 `Pinned`，解除钉住后才可写回或替换。

答题句：

```text
钉住可以避免内存指针悬空，但会降低缓冲区可替换页面数量。
```

## 7. 变长记录、跨块记录与记录修改

### 7.1 变长记录适用场景

适用于：

- 具有可变长度的数据项。
- 具有重复字段。
- 可变记录格式。
- 大量字段或大对象。

常见表示方法：

```text
记录头部 + 字段长度/偏移 + 指向变长字段的指针 + 记录主体
```

课件示例：

- 第一个变长字段可以不需要指针。
- 重复字段可指向一组值或引用表。
- 变长字段可采用间接存取。

### 7.2 XML/JSON 可变记录

XML/JSON 记录通常包含：

- 属性/字段名。
- 字段类型。
- 字段长度。
- 嵌套结构。

结构可通过：

- 自描述 JSON。
- XML Schema/DTD。

### 7.3 跨块记录

当记录太大，不能完全放入一个块中时，需要跨块存储。

块头部可保存：

- 记录片段头部标识。
- 片段所在块的序号。
- 指向下一片段的指针。

### 7.4 BLOB/CLOB 大对象

对于大对象，常将内容存储在多个块上，也可跨多个磁盘存储。

常见类型：

- `BLOB`：二进制大对象。
- `CLOB`：字符大对象。
- `TEXT/IMAGE/VIDEO` 等可作为 BLOB 存储。

### 7.5 记录插入

插入位置选择：

```text
1. 优先在当前块中找足够空闲空间。
2. 若没有足够空间，可使用溢出块。
3. 维护块头部空闲空间目录。
```

若文件按某个键有序：

```text
插入可能导致块分裂或记录迁移。
```

### 7.6 记录删除

删除方式：

| 方式 | 含义 |
| --- | --- |
| 立即删除 | 删除记录并回收块内空间 |
| 快速删除 | 维护块内可用空间链表 |
| 墓碑删除 | 保留删除标记，外部引用仍可找到 |

注意：

```text
若有外部指针引用记录，直接移动或删除记录会导致指针失效。
```

### 7.7 记录更新

定长记录：

```text
存储空间位置不变，内容改变。
```

变长记录：

```text
若变长字段变长，可能无法原地更新。
可采用 DELETE/INSERT 或迁移到新位置。
```

答题比较：

```text
in-place 更新：地址稳定，代价低，但要求空间足够。
replace 更新：可处理变长增长，但可能改变地址，需要更新指针或映射。
```

## 8. 表的组织方式

### 8.1 常见存放方式

课件列出：

```text
1. 堆存放方式
2. 顺序存放方式
3. 多表聚簇存放方式
4. LSM 树存放方式
```

### 8.2 堆表 Heap Table

特点：

- 表中的记录可存放在任意块中。
- 没有顺序要求。
- 插入时找到合适空闲空间即可。
- 若没有足够空闲空间，就为该表申请新块。

适合：

- 插入频繁。
- 对顺序扫描或排序要求不高。

缺点：

- 按条件查找可能需要全表扫描，除非有索引。

### 8.3 顺序存放

特点：

```text
按指定属性或属性组的取值大小顺序存放记录。
```

优点：

- 适合范围查询。
- 适合按排序属性的顺序扫描。

缺点：

- 插入/删除可能导致记录移动。
- 为保持顺序，维护代价较高。

### 8.4 多表聚簇

多表聚簇把经常一起连接访问的多个表的相关记录存放在相邻位置。

优点：

- 减少连接查询时的磁盘 I/O。
- 对常见连接查询效率高。

缺点：

- 同一表单独查询时可能不如独立存放。
- 更新和插入维护复杂。

### 8.5 LSM 树

`LSM` 是 `Log-Structured-Merge Tree`。

核心思想：

```text
先把写入放到内存结构中，积累到一定规模后按层级批量写入磁盘。
```

特点：

- 写入效率高。
- 后台需要合并 `compaction`。
- 读可能需要查多个层级，可配合索引或过滤器优化。

适用：

- 写密集型系统。
- NoSQL 数据库，如 Cassandra、RocksDB、HBase、LevelDB 等。

### 8.6 行存 vs 列存

| 存储方式 | 优势 | 劣势 | 适用场景 |
| --- | --- | --- | --- |
| 行存 | 一行完整记录连续，读写整行方便 | 聚合少数字段时可能读多余数据 | OLTP、事务处理 |
| 列存 | 同一列连续，压缩好，聚合快 | 写入效率低，读取整行较慢 | OLAP、分析查询 |

列存优势：

- 紧缩存储。
- 针对挖掘/分析操作读取更高效。

列存劣势：

- 写低效。
- 记录读高效性较差。

## 9. 元数据组织

### 9.1 元数据定义

元数据是关于数据的数据。

课件示例：

- 关系信息，如关系名、类型、存储信息。
- 属性信息。
- 完整性约束。
- 数据库中定义的索引、视图、触发器、存储过程、函数等。
- 用户名称、密码、授权信息。
- 关于关系和属性的统计信息。

### 9.2 元数据存储

DBMS 通常把元数据存放在数据字典或系统目录中。

在关系数据库中：

```text
数据字典的组织方式通常是关系表。
存放元数据的数据字典表也称为系统表。
```

答题句：

```text
系统表本身也是表，DBMS 通过系统表管理用户表、字段、约束、索引、权限和统计信息。
```

## 10. 缓冲区

### 10.1 为什么设置缓冲区

原因：

```text
1. 提供 DBMS 的设备独立性。
2. 提高存取效率。
3. 让上层操作不直接依赖外存设备变化。
4. 通过预读和延迟写减少 I/O。
```

### 10.2 缓冲区位置

缓冲区在内存中，RDBMS 通常采用该方法。

对比：

- OS 也会管理一部分缓冲。
- DBMS 通常维护自己的缓冲区，以便结合查询、事务和恢复需求。

### 10.3 缓冲区组成

缓冲区主要缓存：

```text
1. 磁盘数据页面
2. 索引
3. 日志等
```

日志可用于故障恢复。

### 10.4 缓冲区读写操作

读数据：

```text
1. 先在缓冲区查找页面。
2. 若命中，直接从缓冲区读取。
3. 若未命中，缓冲区管理器从磁盘读入页面。
4. 若缓冲区满，先选择页面替换。
```

更新数据：

```text
1. 修改缓冲区中的页面。
2. 将页面标记为脏页。
3. 脏页可延迟写回磁盘。
4. 写回时必须遵守日志规则。
```

缓冲区替换：

```text
缓冲区满时，选择一个已缓存页面进行替换。
若被替换页面是脏页，需要先写回磁盘。
```

### 10.5 预写日志 WAL

`WAL` 是 `Write-Ahead Log`。

原则：

```text
在数据页写回磁盘前，相关日志记录必须先写到稳定存储。
```

作用：

- 防止系统崩溃后数据页和日志不一致。
- 支持撤销和重做。
- 保证故障恢复时能判断哪些操作成功或失败。

答题句：

```text
数据库使用预写日志，使对数据页的修改在真正落盘前已经记录到日志，从而在故障后可以根据日志恢复一致状态。
```

### 10.6 页面固定与锁

页面固定 `Pin`：

```text
当进程正在读写某个页面时，该页面不应被替换。
```

共享锁 `S Lock`：

- 多个事务可同时读同一页面。
- 读锁之间兼容。

排他锁 `X Lock`：

- 写页面时独占。
- 其他事务不能同时读写该页面。

### 10.7 页面替换策略

目标：

```text
最大化缓冲区命中率，最小化访问磁盘次数。
```

| 策略 | 全称 | 规则 |
| --- | --- | --- |
| LRU | Least Recently Used | 最近最少使用的页面被替换 |
| LFU | Least Frequently Used | 最不经常使用的页面被替换 |
| LRU-K | 最近 K 次访问策略 | 前第 K 次访问越久远，越应该被替换 |

LRU 口令：

```text
前 1 次使用时间越晚，未来越可能使用，越不应该替换。
```

LFU 口令：

```text
使用频率越高，未来越可能使用，越不应该替换。
```

LRU-K 口令：

```text
前第 K 次访问时间越晚，未来越可能使用，越不应该替换。
```

### 10.8 替换策略计算模板

题型：

```text
给定缓冲区大小和页面访问序列，判断 LRU/LFU 缺页次数或替换过程。
```

步骤：

```text
1. 画表：列为访问序列，行为缓冲帧。
2. 每访问一个页面：
   若已在缓冲区，命中，不读盘。
   若不在缓冲区，缺页，需要读盘。
3. 若缓冲区未满，直接放入空帧。
4. 若缓冲区已满，按策略选择牺牲页。
5. 若牺牲页为脏页，先写回。
6. 统计缺页次数或磁盘读写次数。
```

答题标注：

```text
命中：不需要读磁盘。
缺页：需要读磁盘。
替换脏页：先写回再读入。
```

## 11. 高频题型模板

### 11.1 “计算磁盘访问时间”

模板：

```text
T = Tseek + Trotate + Ttran
Ttran = 数据块大小 / 传输速率
总时间 = 各次访问时间之和
```

答题格式：

```text
第 i 次访问：
移动距离 = ...
Tseek = ...
Trotate = ...
Ttran = ...
Ti = ...
累计完成时间 = ...
```

### 11.2 “用电梯算法给出服务顺序”

模板：

```text
1. 初始磁头位置为 ...
2. 当前方向为 ...
3. 已到达请求集合为 ...
4. 沿当前方向依次访问：...
5. 到达端点或无请求后换向：...
6. 最终服务顺序为：...
```

### 11.3 “RAID 校验/恢复”

模板：

```text
校验块 P = D1 XOR D2 XOR ... XOR Dn
若 Di 丢失：
Di = P XOR D1 XOR ... XOR D(i-1) XOR D(i+1) XOR ... XOR Dn
```

结论句：

```text
因此 RAID4/5 可通过异或校验恢复单盘故障；RAID6 通过双校验提高多盘故障容忍能力。
```

### 11.4 “说明 DBMS 页式存储结构”

模板：

```text
DBMS 通常把数据库组织为：
数据库 -> 表空间 -> 段 -> 分区 -> 数据块。
其中数据块是 DBMS 与磁盘之间 I/O 的基本单位，
记录存放在数据块中，块头部维护记录目录和空闲空间信息。
```

### 11.5 “比较定长记录和变长记录”

模板：

```text
定长记录：
字段长度固定，记录长度固定，定位和更新方便，但可能浪费空间。

变长记录：
适合可变字段、重复字段和大字段，需要记录头、长度、偏移或指针，
空间利用率更高，但定位和更新复杂。
```

### 11.6 “说明逻辑地址到物理地址转换”

模板：

```text
客户端或上层使用逻辑地址；
DBMS 通过映射表把逻辑地址转换为物理地址；
物理地址包含设备、柱面、磁道、块号和块内偏移。
记录移动时只需要修改映射表，外部逻辑地址可以保持不变。
```

### 11.7 “说明记录插入/删除/更新过程”

模板：

```text
插入：
先在块中查找足够空闲空间；若无空间，则使用溢出块或申请新块。

删除：
可立即回收空间，也可用墓碑标记保留地址稳定性。

更新：
定长记录可原地更新；变长记录若变长，可能需要迁移或 DELETE/INSERT。
```

### 11.8 “比较堆表、顺序存放、多表聚簇、LSM”

| 方式 | 优点 | 缺点 | 适用 |
| --- | --- | --- | --- |
| 堆表 | 插入简单 | 查找可能全表扫描 | 插入多、无顺序要求 |
| 顺序存放 | 范围查询快 | 插入维护复杂 | 按某列范围查询多 |
| 多表聚簇 | 连接查询少 I/O | 单表访问和更新复杂 | 常做固定连接 |
| LSM 树 | 写入快 | 读可能查多层，需合并 | 写密集系统 |

### 11.9 “说明缓冲区读写与 WAL”

模板：

```text
读页面时先查缓冲区，命中则直接读；未命中才从磁盘读入。
更新页面时先修改缓冲区并标记为脏页，脏页可延迟写回。
为了故障恢复，数据页写回磁盘前必须先写日志，即预写日志 WAL。
```

### 11.10 “计算 LRU/LFU 页面替换”

模板：

```text
1. 画缓冲帧表。
2. 逐个处理访问序列。
3. 命中则状态不变或更新时间/频率。
4. 缺页且有空位则装入。
5. 缺页且无空位则按策略替换：
   LRU：替换最近最久未使用页。
   LFU：替换使用次数最少页。
6. 统计缺页次数和磁盘读写次数。
```

## 12. 易错清单

- 磁盘访问时间不是只有传输时间，通常寻道和旋转延迟更关键。
- HDD 顺序访问远快于随机访问，SSD 随机访问优势明显。
- FCFS 简单但可能导致磁头长距离移动。
- 电梯算法要考虑请求到达时间，未到达请求不能提前服务。
- RAID 可以提高可靠性，但不能代替备份和日志恢复。
- RAID4 有固定校验盘瓶颈，RAID5 将校验分布到各盘。
- 数据块是 DBMS I/O 基本单位，不是单条记录。
- 记录地址通常由块 ID 和块内偏移组成。
- 变长记录更新可能导致记录迁移，不能总是假设原地更新。
- 删除记录若有外部指针引用，直接移动记录可能破坏引用。
- 元数据是关于数据的数据，通常存放在数据字典/系统表中。
- 缓冲区命中不需要磁盘 I/O，缺页才需要读盘。
- 脏页替换前必须写回；WAL 要求数据页落盘前日志先落盘。
- LRU 看“最近使用时间”，LFU 看“使用频率”，LRU-K 看“前第 K 次访问时间”。
